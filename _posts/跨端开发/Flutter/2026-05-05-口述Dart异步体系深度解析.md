## 一句话概括

Dart 异步体系的四个层次：Event Loop（发动机）→ Future/async-await（方向盘）→ Stream（传送带）→ Isolate（分身术），用同步写法做异步的事，用独立堆内存做真正的并行，是 Dart 在安全性和性能之间最精妙的平衡。

## 核心知识点

### 1. 四层架构全景

```
┌─────────────────────────────────────────┐
│           Dart 异步体系四层模型            │
├─────────────────────────────────────────┤
│ L4 Isolate   │ 分身术：多个独立堆+事件循环  │
│ L3 Stream    │ 传送带：多次数据推送         │
│ L2 Future    │ 方向盘：单次异步结果         │
│ L1 EventLoop │ 发动机：微任务+事件队列      │
└─────────────────────────────────────────┘
```

面试时这样回答：第一层 Event Loop 是基础引擎，第二层 Future 是一个值的异步抽象，第三层 Stream 是多个值的异步序列，第四层 Isolate 是真正的多核并行。四层从单线程异步到多核并行，覆盖了所有并发场景。

### 2. Event Loop：微任务 vs 事件队列对比表

| 维度 | 微任务 | 事件队列 |
|------|--------|---------|
| 来源 | `scheduleMicrotask`、`Future.then`、`async` 恢复点 | I/O、Timer、用户输入 |
| 优先级 | 高于事件队列 | 微任务全部清空后才处理 |
| 适合 | 状态通知（小、快） | 实际异步工作 |
| 危险 | 递归注册微任务会饿死事件队列 | — |

**一条铁律**：Dart 永远不会中断当前同步代码去执行微任务或事件队列。同步代码跑完→清空微任务→取一个事件→下一轮。

### 3. Future ↔ Stream ↔ Isolate 选型速查

```dart
// 场景1：一次网络请求 → Future
http.get(url).then((res) => jsonDecode(res.body));

// 场景2：WebSocket 持续推送 → Stream
webSocket.listen((data) => print('新消息: $data'));

// 场景3：50MB JSON 解析卡 UI → compute（Isolate）
compute(jsonDecode, hugeJsonString);

// 场景4：Firebase 实时数据库 → Stream
FirebaseFirestore.instance.collection('users').snapshots();
```

选型口诀：**一次 → Future，多次 → Stream，耗 CPU → Isolate**。

### 4. async/await 不是"飞行模式"——它是状态机

每次 await = 一个挂起点 = 编译器生成一个 case。函数暂停后事件循环继续处理其他任务（UI 事件、动画帧等）。await 后面的代码是"回调"，不是"下一行"。

```dart
// 这是你看到的样子
final user = await api.getUser();  // 挂起
print(user.name);                  // 恢复后执行

// 这是编译器的理解
api.getUser().then((user) {
  print(user.name); // 回调
});
```

### 5. Zone：错误传播的最后一道防线

```dart
runZonedGuarded(() {
  Future.error('异步错误');
}, (error, stack) {
  print('Zone 捕获: $error');
});
```

Zone 是"异步调用栈"的概念——Future 回调中抛出的错误，try/catch 无法捕获，但 Zone 能。Flutter 的全局错误处理就是基于 Zone 实现的。

## 其实你每天都在用

- **`onPressed: () => setState(...)`** → 同步回调，注意 async 陷阱
- **`StreamBuilder`** → Stream 到 UI 的自动桥接
- **`compute()`** → Flutter 的 Isolate 快捷方式
- **`runZonedGuarded`** → Flutter 全局异常捕获底层的底层
- **`Timer.periodic` / `Future.delayed`** → 事件队列中的定时器

## 常见误解（FAQ）

**❌ 误区：「Dart 是单线程，所以性能不如多线程语言」**

单线程只限于**单个 Isolate 内**。你可以开多个 Isolate 利用多核。而且单线程内无锁，省去了线程切换和竞争开销，对于 IO 密集型任务反而更高效。

**❌ 误区：「Stream 就是多了几个 then 的 Future」**

本质不同。Future 是"推一次"，Stream 是"推 N 次"。更关键的是，Stream 支持背压控制（pause/resume），Future 不行。

**❌ 误区：「所有 async 函数都应该在调用处加 await」**

不是。`Future` 类型本身是一个对象，你不 await 它它照样跑——这叫"fire and forget"。如果结果不重要（如埋点上报），不加 await 反而更高效。

## 一句话总结

L1-L4 四层模型是你理解 Dart 异步的"二进制"——不用背，理解层的递进关系就自然记住了。Event Loop 跑引擎，Future 管单个，Stream 管多个，Isolate 管并行，Zone 兜底错误。
