## 一句话概括

Future 是 Dart 异步编程的基石——它代表"将来某个时刻会完成的计算"，通过事件循环调度和 then 链式调用，让单线程能处理成千上万个并发操作而不卡 UI。

## 核心知识点

### 1. Future 的三种状态：不可逆的状态机

```dart
final f = Future(() => 'hello');
// 此时 f 是 Uncompleted
f.then((v) => print(v)); // 注册回调
// 某时刻后 f 变为 Completed(value: 'hello')，触发回调

// 这是面试高频题：Future 状态一旦从 uncompleted → completed，不可回退
```

三种状态：Uncompleted（待定）→ Completed with **value** 或 Completed with **error**。不会从"完成"回到"待定"。

### 2. 创建方式：6 种，面试至少知道 3 种

```dart
// 1. Future() —— 调度到事件队列
Future(() => 42);

// 2. Future.value() —— 立即可用的已完成 Future（微任务触发回调）
Future.value('done').then(print);  // 比 Future(()=>'done') 快

// 3. Future.delayed() —— 延迟执行
Future.delayed(Duration(seconds: 2), () => 'wake up');

// 4. Future.error() —— 已完成（错误态）
Future.error(Exception('fail'));

// 5. Future.sync() —— 立即同步执行函数体，then 回调进入微任务
Future.sync(() => 42); // 函数体同步跑，回调异步

// 6. Completer —— 手动控制完成时机
final c = Completer<String>();
c.future.then(print);
c.complete('manual'); // 外部控制何时完成
```

`Future.value` vs `Future()`：`value` 的 then 回调在**微任务**中执行（比事件队列优先），`Future()` 在**事件队列**中执行。

### 3. then 链：扁平化是关键

```dart
Future.value('start')
  .then((v) => v.length)          // Future<String> → Future<int>
  .then((v) => Future.value(v*2)) // 返回 Future 会自动"扁平化"——不会变成 Future<Future<int>>
  .then((v) => print(v));         // v 是 int，不是 Future<int>
```

then 回调返回 `Future<T>` 时会被**扁平化**（flattening）——你拿到的永远是解包后的值。这是 Dart 的类型安全设计。

### 4. 错误处理：catchError 拦截，错误会沿链传播

```dart
Future.error('oops')
  .then((v) => print('不会执行'))
  .catchError((e) => print('捕获: $e'))       // 拦截
  .then((v) => print('恢复后继续: $v'));       // 打印 "恢复后继续: null"

// test 参数：只捕获特定类型的错误
Future.error(FormatException('bad'))
  .catchError((e) => print('格式错误'), test: (e) => e is FormatException);
```

错误沿 then 链**向下传播**，直到被 `catchError` 拦截。拦截后可以返回"恢复值"继续链，或者 rethrow 让它继续传播。

### 5. 并发控制：await 不等于串行

```dart
// ❌ 串行：总共 5 秒
final u = await fetchUser();    // 等 3 秒
final p = await fetchPosts();   // 再等 2 秒

// ✅ 并行：总共 3 秒
final [u2, p2] = await Future.wait([
  fetchUser(),
  fetchPosts(),
]);

// 🏆 谁快返回谁
final fastest = await Future.any([
  fetchUser(), fetchPosts(),
]); // 取先完成的
```

`await` 只是**暂停当前函数**，不阻塞线程。理解这一点才能写出"看着像串行但实际并行"的代码。

## 其实你每天都在用

- **`http.get(url)` 返回 `Future<Response>`**：网络请求全在 Future 里，不然 UI 线程会卡死
- **`SharedPreferences.getInstance()`**：初始化 SharedPreferences 是异步的，返回 Future
- **`Navigator.push(context, route)`**：push 返回 `Future<T?>?`，等页面 pop 时才完成
- **`showDialog()` 返回 Future**：对话框关闭时 resolve，拿到用户的选择结果
- **`Image.network(url)` 内部用 Future 加载图片**：占位图先显示，加载完成后替换

## 常见误解（FAQ）

**❌ 误区：「await 会阻塞 UI 线程」**

不阻塞。`await` 只暂停当前 async 函数，事件循环继续处理 UI 事件、动画帧等。这就是 Dart 单线程模型能跑 Flutter 流畅 60fps 的秘密。

**❌ 误区：「`Future.wait` 会取消未完成的 Future」**

不会。`Future.wait` 里的所有 Future 都会执行到结束，即使其中一个报错，其他的也继续运行。需要取消要用 `CancelableOperation` 或手动管理。

**❌ 误区：「then 回调里的代码和 await 后面的代码执行时机一样」**

不完全一样。`Future.value(1).then(print)` 的回调在**微任务**中执行，`Future(() => 1)` 在**事件队列**中执行。微任务优先级更高。这是面试官最爱追问的细节。

**❌ 误区：「async 函数总是异步的」**

async 函数在**第一个 await 之前**是同步执行的。`loadData() async { print('sync'); await delay(); }` 中 `print('sync')` 是同步输出。
