## 一句话概括

数组扁平化（Array Flatten）将多维嵌套数组转为单层数组，面试考的是你能不能理解递归、栈和 Generator 三种不同思路的取舍。

## 核心知识点

### 1. 递归实现 — 最直观

```javascript
function flatten(arr) {
  let result = [];
  for (const item of arr) {
    Array.isArray(item) ? result.push(...flatten(item)) : result.push(item);
  }
  return result;
}

flatten([1, [2, [3, 4]]]); // [1, 2, 3, 4]
```

**优点：** 语义清晰，面试第一反应。**缺点：** 嵌套过深会栈溢出（Chrome 约 10000 层）。

### 2. reduce 函数式写法 — 一行秒杀

```javascript
const flatten = arr => arr.reduce((acc, cur) =>
  acc.concat(Array.isArray(cur) ? flatten(cur) : cur), []);

// 本质和递归一样，只是表达方式不同
```

### 3. 栈实现 — 非递归，不爆栈

```javascript
function flatten(arr) {
  const stack = [...arr];
  const result = [];
  while (stack.length) {
    const item = stack.pop();
    Array.isArray(item) ? stack.push(...item) : result.push(item);
  }
  return result.reverse(); // 栈是 LIFO，需要反转
}

flatten([1, [2, [3, 4]]]); // [1, 2, 3, 4]
```

**核心思路：** 用数组模拟调用栈，每次从栈顶取元素，是数组就展开后压回去，不是就加入结果集。

### 4. Generator 实现 — 惰性求值

```javascript
function* flatten(arr) {
  for (const item of arr) {
    Array.isArray(item) ? yield* flatten(item) : yield item;
  }
}

// 逐个消费，不用一次性全部加载
for (const val of flatten([1, [2, [3]]])) {
  console.log(val); // 1, 2, 3
}

// 也可以一把转为数组
[...flatten([1, [2, [3]]])]; // [1, 2, 3]
```

**适用场景：** 数据量巨大、边读边处理，不需要一次性在内存里铺开整个结果。

### 5. 指定深度扁平化

```javascript
function flatten(arr, depth = Infinity) {
  return depth > 0
    ? arr.reduce((acc, cur) =>
        acc.concat(Array.isArray(cur) ? flatten(cur, depth - 1) : cur), [])
    : arr.slice();
}

const data = [1, [2, [3, [4]]]];
flatten(data, 1); // [1, 2, [3, [4]]]
flatten(data, 2); // [1, 2, 3, [4]]
```

**面试提醒：** ES2019 原生 `arr.flat(depth)` 直接干这个事——但面试时你得能手写，并且能讲出递归 vs 栈的取舍。

## 其实你每天都在用

- **API 返回的嵌套结构**：后端给的 `[{ id:1, children:[2,3] }, [4, [5]]]`，展成 ID 列表传给表格组件
- **树形目录 → 路径列表**：文件树的嵌套节点展平后生成面包屑导航
- **路由配置**：多层嵌套路由展平后批量注册到 Vue Router / React Router
- **批量数据处理**：从多个不同格式的数据源提取所有记录，先 flat 再统一处理
- **DOM 操作**：`element.children` 是嵌套的，展平后方便全量遍历

## 常见误解

- **❌ 误区：「递归是万能方案」** 生产数据嵌套层级不确定时，递归可能导致栈溢出。大数据量或深层嵌套优先用栈或 Generator。

- **❌ 误区：「`arr.flat(Infinity)` 就够了」** 语法上够了，但面试考察的是原理。而且 `flat` 是同步的一次性展开，不支持惰性消费和中间处理。

- **❌ 误区：「栈实现里 `reverse()` 是多余的」** 不是。栈是 LIFO（后进先出），最后一次 pop 的是原数组第一个元素，所以结果需要反转。如果想省 `reverse`，用 `shift()` + `unshift()` 把栈当队列用——但这样时间复杂度会从 O(n) 变成 O(n²)。

- **❌ 误区：「`yield* flatten(item)` 就是递归调用」** 形式上像递归，但 Generator 不会建调用栈。`yield*` 是把子 Generator 的产出逐个转发，本质是迭代，不存在递归深度的物理限制。

## 一句话总结

扁平化的路数很多，面试时先写递归表达思路，再补上栈实现表现深度，最后提一句 Generator 的惰性求值场景——三件套齐活儿。
