## 一句话概括

数组扁平化就是从 `[1, [2, [3, 4]]]` 到 `[1, 2, 3, 4]`。面试官不关心你知道 `arr.flat()`——考的是递归爆栈的边界意识、栈实现为何要 `reverse()`、Generator 的惰性求值思维。

## 核心知识点

### 1. 递归 — 最直观，但有软肋

```javascript
function flatten(arr) {
  const result = [];
  for (const item of arr) {
    Array.isArray(item) ? result.push(...flatten(item)) : result.push(item);
  }
  return result;
}

// 函数式写法：
const flatten = arr =>
  arr.reduce((acc, cur) => acc.concat(Array.isArray(cur) ? flatten(cur) : cur), []);

flatten([1, [2, [3, [4]]]]); // [1, 2, 3, 4]
```

**优点：** 代码最短。**致命伤：** 嵌套层级超过 ~10000 直接爆栈。递归深度 = 嵌套层数，浏览器调用栈有上限（Chrome ~10000）。

### 2. 栈实现 — 不爆栈的稳健方案

```javascript
function flatten(arr) {
  const stack = [...arr];
  const result = [];
  while (stack.length) {
    const next = stack.pop();
    Array.isArray(next) ? stack.push(...next) : result.push(next);
  }
  return result.reverse(); // ← 面试官等你说这句
}
```

**为什么需要 `reverse()`：** 用 `pop()` 从栈顶取 → 子数组展开后被 `push` 回去 → 原来的顺序被反转。`[1, [2, [3]]]` 的处理轨迹：

```
初始栈: [1, [2,[3]]]
pop [2,[3]] → push 2, [3]   → 栈: [1, 2, [3]]
pop [3]     → push 3       → 栈: [1, 2, 3]
pop 3 → result: [3]       → pop 2 → [3,2] → pop 1 → [3,2,1]
最后 reverse → [1, 2, 3] ✅
```

**能不能不用 reverse？** 用 `shift()` 替代 `pop()` 把栈当队列——但 shift 是 O(n)，整体变 O(n²)。

### 3. Generator — 惰性求值，内存友好

```javascript
function* flatten(arr) {
  for (const item of arr) {
    Array.isArray(item) ? yield* flatten(item) : yield item;
  }
}

// 惰性消费：一个个吐，不一次性展开（大数组关键优势）
for (const val of flatten(hugeArray)) {
  process(val); // 内存中同时只存在当前一个值
}

// 也可以一次性收集
const all = [...flatten([1, [2, [3]]])]; // [1, 2, 3]
```

**`yield*` 不是递归调用：** Generator 的 `yield*` 把子 Generator 的产出值逐个向上转发，不会增长调用栈。这就是为什么 Generator 不会爆栈。

### 4. 指定深度

```javascript
function flatten(arr, depth = Infinity) {
  return depth > 0
    ? arr.reduce((acc, cur) =>
        acc.concat(Array.isArray(cur) ? flatten(cur, depth - 1) : cur), [])
    : arr.slice();
}

flatten([1, [2, [3, [4]]]], 1); // [1, 2, [3, [4]]]
flatten([1, [2, [3, [4]]]], 2); // [1, 2, 3, [4]]
```

### 5. 三方案对比

| | 递归 | 栈 | Generator |
|---|---|---|---|
| 代码量 | 最少 | 中等 | 中等 |
| 爆栈风险 | ✅ 有（万层） | ❌ 无 | ❌ 无 |
| 惰性求值 | ❌ | ❌ | ✅ |
| 可指定深度 | 容易 | 需额外跟踪 | 容易 |

## 其实你每天都在用

- **树形接口数据展平**：后端返回 `[{ id:1, children:[{id:2}] }]`，扁平后直接丢进 `<table>` 渲染
- **路由配置递归注册**：Vue Router 嵌套路由扁平后批量 `router.addRoute`
- **文件目录 → 路径列表**：树形文件夹展平生成面包屑或侧边栏
- **多数据源合流**：三个 API 各自返回不同层级的数组，先 flat 再统一 map/filter
- **DOM 深遍历**：`element.children` 是嵌套的，展平后统一绑事件

## 常见误解

- **❌ 误区：「递归是最佳方案」** 生产场景嵌套深度不可控时（AST 遍历、爬虫数据、深度嵌套的 JSON），递归随时爆栈。大数据量用栈，惰性消费用 Generator。

- **❌ 误区：「`arr.flat(Infinity)` 就够了为什么要手写」** 面试考原理。而且 `flat()` 是同步一次性展开——十万条数据要边展平边处理时 Generator 才是正解。

- **❌ 误区：「栈实现里 `reverse()` 浪费性能」** `reverse()` O(n)，flatten 本身也 O(n)，同量级不敢说浪费。用 `shift()` 避免 reverse 才是真正的性能杀手：shift 每次把后续元素前移一位 → O(n²)。

- **❌ 误区：「`Array.isArray` 就够了」** `arguments`、`NodeList`、`FileList` 看起来像数组但 `Array.isArray` 返回 `false`。要覆盖这些需 `Array.from()` 或 `[...likeArray]` 先转换。

## 一句话总结

面试三步走：递归说清楚思路 → 栈实现讲明白为什么 `reverse()` → Generator 提一嘴惰性求值——三件套打完，这道题就毕业了。
