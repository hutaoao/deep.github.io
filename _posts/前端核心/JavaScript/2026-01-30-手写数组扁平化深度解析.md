## 一句话概括

把 `[1, [2, [3, 4]]]` 变成 `[1, 2, 3, 4]`。面试官不关心你知道 `arr.flat()`——考的是递归爆栈意识、栈实现为什么必须 `reverse()`、Generator 惰性求值的工程价值。

## 核心知识点

### 1. 递归——最直观，但有硬伤

```js
const flatten = arr =>
  arr.reduce((acc, cur) =>
    acc.concat(Array.isArray(cur) ? flatten(cur) : cur), []
  );

flatten([1, [2, [3, [4]]]]); // [1, 2, 3, 4]
```

**优点：** 3 行搞定。**致命伤：** 嵌套深度 ≈ 递归深度，Chrome 调用栈上限 ~10000 层，超过直接 `RangeError: Maximum call stack size exceeded`。AST 遍历、深度嵌套的 JSON 都踩过这个坑。

### 2. 栈实现——不爆栈，但有个坑

```js
function flatten(arr) {
  const stack = [...arr];
  const result = [];
  while (stack.length) {
    const item = stack.pop();
    if (Array.isArray(item)) {
      stack.push(...item);
    } else {
      result.push(item);
    }
  }
  return result.reverse(); // ⚠️ 面试官等的就是这句——为什么必须 reverse？
}

// 推演 [1, [2, [3]]]：
// stack: [1, [2,[3]]] → pop [2,[3]], push 2,[3]  → [1, 2, [3]]
// pop [3], push 3                            → [1, 2, 3]
// pop 3,2,1 → result: [3, 2, 1]
// reverse → [1, 2, 3] ✅
```

**为什么 `reverse()` 省不了：** `pop()` 取最后一个 + `push()` 加到末尾 → 子数组展开后和原顺序反了。用 `shift()` 代替 `pop()` 确实不需要 reverse，但 shift 是 O(n)（所有元素前移），整体变 O(n²)。

### 3. Generator——惰性求值，大数组首选

```js
function* flatten(arr) {
  for (const item of arr) {
    if (Array.isArray(item)) {
      yield* flatten(item); // yield* 不增加调用栈，不会爆
    } else {
      yield item;
    }
  }
}

// 惰性消费：内存里同时只存在当前这一项
for (const val of flatten(hugeArray)) {
  process(val); // 百万条数据也不会同时加载到内存
}

// 一次性收集也行
[...flatten([1, [2, [3]]])]; // [1, 2, 3]
```

**`yield*` 不是递归：** 它把子 Generator 的产出逐个转发，不增长调用栈——这就是 Generator 不会爆栈的根本原因。

### 4. 指定深度展平

```js
const flatten = (arr, depth = Infinity) =>
  depth > 0
    ? arr.reduce((acc, cur) =>
        acc.concat(Array.isArray(cur) ? flatten(cur, depth - 1) : cur), []
      )
    : arr.slice();

flatten([1, [2, [3, [4]]]], 1); // [1, 2, [3, [4]]]
flatten([1, [2, [3, [4]]]], 2); // [1, 2, 3, [4]]
```

### 5. 方案对比

| | 递归 | 栈 | Generator |
|---|---|---|---|
| 爆栈风险 | ✅ 有（万层） | ❌ | ❌ |
| 惰性求值 | ❌ | ❌ | ✅ |
| 指定深度 | 容易 | 需额外跟踪 | 容易 |
| 内存占用 | 高（全量） | 高（全量） | 低（按需） |

## 其实你每天都在用

- **树形数据展平渲染**：后端返回 `[{id:1, children:[{id:2}]}]` 的菜单/目录，flatten 后直接丢进表格或虚拟列表
- **Vue Router 动态注册**：权限接口返回嵌套路由，flatten 后 `router.addRoute()` 批量注册
- **文件树 → 面包屑**：树形文件夹展平成路径列表，遍历生成导航面包屑
- **多数据源合流**：三个 API 各自返回不同嵌套层级的数组，先 flat 再统一 `map`/`filter`
- **DOM 深层遍历**：`element.children` 嵌套深，展平后统一绑定事件或查找节点

## 常见误解

- **❌ 误区：「递归是最佳方案，代码最短」** 生产环境嵌套深度不可控时（AST 遍历、爬虫数据、深度 JSON），递归随时爆栈。大数据量用栈，惰性消费用 Generator。

- **❌ 误区：「`arr.flat(Infinity)` 搞定，手写没意义」** 面试考的是原理。而且 `flat()` 是同步一次性展开——如果你有十万条数据要边展平边处理（流式），Generator 才是正解。

- **❌ 误区：「`reverse()` 浪费性能，不如不用」** `reverse()` 是 O(n)，flatten 本身也 O(n)，同量级。真正要命的是用 `shift()` 来避免 reverse——shift 每次把所有元素前移一位，O(n) 套在 O(n) 循环里就是 O(n²)。

- **❌ 误区：「`Array.isArray` 判断数组就够了」** `arguments`、`NodeList`、`FileList` 等类数组对象 `Array.isArray` 返回 `false`，但迭代行为类似数组。需要兼容时用 `Array.from()` 先转。

## 一句话总结

面试三件套：递归说清思路 → 栈实现讲透 `reverse()` 原因 → Generator 提一嘴惰性求值。打完这三张牌，这道题就毕业了。
