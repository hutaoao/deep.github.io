## 一句话概括

手写 Promise 不是去"发明轮子"，而是实现 **Promise/A+ 规范**（then/catch/状态管理等）——这能加深对 JS 异步机制的理解，也是**中级前端面试手写题的经典题目**。

## 背景与意义

### 为什么面试官喜欢考手写 Promise？

手写 Promise 考察的是**多个维度的能力**：

```
1. 对异步编程的理解（回调 vs Promise vs async/await）
2. 对设计模式的理解（状态模式、观察者模式）
3. 对 Promise/A+ 规范的熟悉程度
4. 对 JavaScript 语言特性的掌握（this、闭包、微任务）
5. 代码组织能力（边界条件处理）
```

### 面试高频信号

手写 Promise 在面试中**出现率极高**，且层层递进：
- "实现一个简单的 Promise" → 基础版
- "实现 `.then()` 的链式调用" → 核心
- "实现 `.all()`、`.race()`、`.any()`" → 静态方法
- "Promise 是宏任务还是微任务？如何实现微任务调度？" → 深度

## 概念与定义

### Promise/A+ 规范的核心

Promise 是一个**拥有三种状态**的对象：

```
状态机：
                  ┌────────────┐
                  │  Pending   │
                  │  (等待)     │
                  └─────┬──────┘
                        │
            ┌───────────┼───────────┐
            ▼                       ▼
    ┌──────────────┐       ┌──────────────┐
    │  Fulfilled   │       │   Rejected   │
    │  (已完成)     │       │  (已拒绝)     │
    │  value       │       │  reason      │
    └──────────────┘       └──────────────┘

★ 关键规则：
1. 状态只能从 Pending → Fulfilled（不可逆）
2. 状态只能从 Pending → Rejected（不可逆）
3. 一旦状态改变，就不能再变
```

### then 方法的规则

```
promise.then(onFulfilled, onRejected)

onFulfilled ← 当 promise 变为 fulfilled 时调用
onRejected  ← 当 promise 变为 rejected 时调用

★ 关键：
1. then 返回一个新的 Promise（链式的基石）
2. 回调的返回值决定返回的 Promise 的状态
3. 回调可以返回普通值（包装为 resolved）或 Promise
4. then 的回调是异步调用的（微任务）
```

## 最小示例

```javascript
// 使用标准的 Promise
const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve('成功'), 1000);
});

p.then(
  value => console.log(value),    // 1 秒后输出: "成功"
  reason => console.error(reason) // 不会执行
);
```

## 核心实现拆解

### 第一步：基础构造函数和状态管理

```javascript
class MyPromise {
  // Promise 的三个状态
  static PENDING = 'pending';
  static FULFILLED = 'fulfilled';
  static REJECTED = 'rejected';

  constructor(executor) {
    // 初始状态
    this.state = MyPromise.PENDING;
    this.value = undefined;   // 成功的结果
    this.reason = undefined;  // 失败的原因

    // 用于存储异步场景的回调
    this.onFulfilledCallbacks = [];
    this.onRejectedCallbacks = [];

    // resolve 和 reject 函数
    const resolve = (value) => {
      if (this.state !== MyPromise.PENDING) return; // 状态不可逆
      this.state = MyPromise.FULFILLED;
      this.value = value;
      // 调用所有已存储的成功回调
      this.onFulfilledCallbacks.forEach(fn => fn(value));
    };

    const reject = (reason) => {
      if (this.state !== MyPromise.PENDING) return; // 状态不可逆
      this.state = MyPromise.REJECTED;
      this.reason = reason;
      // 调用所有已存储的失败回调
      this.onRejectedCallbacks.forEach(fn => fn(reason));
    };

    try {
      // 立即执行 executor
      executor(resolve, reject);
    } catch (err) {
      reject(err);
    }
  }

  // then 方法（待完善）
  then(onFulfilled, onRejected) {
    // 先忽略链式调用，只做基础实现
    if (this.state === MyPromise.FULFILLED) {
      typeof onFulfilled === 'function' && onFulfilled(this.value);
    }
    if (this.state === MyPromise.REJECTED) {
      typeof onRejected === 'function' && onRejected(this.reason);
    }
    // 如果还是 Pending（异步还没完成）
    if (this.state === MyPromise.PENDING) {
      this.onFulfilledCallbacks.push(onFulfilled);
      this.onRejectedCallbacks.push(onRejected);
    }
  }
}
```

```javascript
// 测试基础版
const p = new MyPromise((resolve) => {
  setTimeout(() => resolve('OK'), 100);
});

p.then(value => console.log(value));  // 1 秒后输出: OK
```

### 第二步：完整的 then 方法（链式调用）

**这是手写 Promise 最复杂的部分。**

```javascript
class MyPromise {
  // ... 前面的代码

  then(onFulfilled, onRejected) {
    // ★ 值穿透：如果 onFulfilled/onRejected 不是函数，提供一个默认值
    onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : value => value;
    onRejected = typeof onRejected === 'function' ? onRejected : reason => { throw reason; };

    // ★ 返回一个新的 Promise（链式调用的关键）
    const promise2 = new MyPromise((resolve, reject) => {
      // 已经 fulfilled
      if (this.state === MyPromise.FULFILLED) {
        // ★ 异步执行回调（模拟微任务）
        queueMicrotask(() => {
          try {
            const x = onFulfilled(this.value);
            this.resolvePromise(promise2, x, resolve, reject);
          } catch (err) {
            reject(err);
          }
        });
      }

      // 已经 rejected
      if (this.state === MyPromise.REJECTED) {
        queueMicrotask(() => {
          try {
            const x = onRejected(this.reason);
            this.resolvePromise(promise2, x, resolve, reject);
          } catch (err) {
            reject(err);
          }
        });
      }

      // 还是 Pending（异步场景）
      if (this.state === MyPromise.PENDING) {
        this.onFulfilledCallbacks.push(() => {
          queueMicrotask(() => {
            try {
              const x = onFulfilled(this.value);
              this.resolvePromise(promise2, x, resolve, reject);
            } catch (err) {
              reject(err);
            }
          });
        });
        this.onRejectedCallbacks.push(() => {
          queueMicrotask(() => {
            try {
              const x = onRejected(this.reason);
              this.resolvePromise(promise2, x, resolve, reject);
            } catch (err) {
              reject(err);
            }
          });
        });
      }
    });

    return promise2;
  }

  // ★ 处理 then 回调的返回值
  resolvePromise(promise2, x, resolve, reject) {
    // 防止死循环：promise2 不能和 x 是同一个对象
    if (promise2 === x) {
      reject(new TypeError('Chaining cycle detected for promise'));
      return;
    }

    // 如果 x 是 MyPromise 实例
    if (x instanceof MyPromise) {
      x.then(
        value => resolve(value),
        reason => reject(reason)
      );
      return;
    }

    // 如果 x 是 thenable（有 then 方法的对象/函数）
    // 比如：{ then: (resolve) => resolve(42) }
    if (x !== null && (typeof x === 'object' || typeof x === 'function')) {
      try {
        const then = x.then;
        if (typeof then === 'function') {
          // 是 thenable，调用它的 then
          let called = false;  // ★ 防止多次调用
          try {
            then.call(
              x,
              value => {
                if (called) return;
                called = true;
                this.resolvePromise(promise2, value, resolve, reject);
              },
              reason => {
                if (called) return;
                called = true;
                reject(reason);
              }
            );
          } catch (err) {
            if (!called) reject(err);
          }
        } else {
          // 普通对象
          resolve(x);
        }
      } catch (err) {
        reject(err);
      }
    } else {
      // 普通值
      resolve(x);
    }
  }
}

// ★ catch 方法（语法糖）
catch(onRejected) {
  return this.then(null, onRejected);
}

// ★ finally 方法
finally(callback) {
  return this.then(
    value => MyPromise.resolve(callback()).then(() => value),
    reason => MyPromise.resolve(callback()).then(() => { throw reason; })
  );
}
```

### 第三步：静态方法

```javascript
class MyPromise {
  // ...

  // Promise.resolve
  static resolve(value) {
    if (value instanceof MyPromise) {
      return value;
    }
    return new MyPromise(resolve => resolve(value));
  }

  // Promise.reject
  static reject(reason) {
    return new MyPromise((_, reject) => reject(reason));
  }

  // Promise.all（全部成功才成功，一个失败就失败）
  static all(promises) {
    return new MyPromise((resolve, reject) => {
      const results = [];
      let completed = 0;

      if (promises.length === 0) {
        resolve(results);
        return;
      }

      promises.forEach((promise, index) => {
        MyPromise.resolve(promise).then(
          value => {
            results[index] = value;  // ★ 保持顺序
            completed++;
            if (completed === promises.length) {
              resolve(results);
            }
          },
          reason => reject(reason)  // 任何一个失败就整体失败
        );
      });
    });
  }

  // Promise.allSettled（所有都执行完，不关心成功/失败）
  static allSettled(promises) {
    return new MyPromise((resolve) => {
      const results = [];
      let completed = 0;

      if (promises.length === 0) {
        resolve(results);
        return;
      }

      promises.forEach((promise, index) => {
        MyPromise.resolve(promise).then(
          value => {
            results[index] = { status: 'fulfilled', value };
            completed++;
            if (completed === promises.length) resolve(results);
          },
          reason => {
            results[index] = { status: 'rejected', reason };
            completed++;
            if (completed === promises.length) resolve(results);
          }
        );
      });
    });
  }

  // Promise.race（第一个完成的胜出）
  static race(promises) {
    return new MyPromise((resolve, reject) => {
      promises.forEach(promise => {
        MyPromise.resolve(promise).then(resolve, reject);
      });
    });
  }

  // Promise.any（第一个成功的胜出，全部失败才报 AggregateError）
  static any(promises) {
    return new MyPromise((resolve, reject) => {
      const errors = [];
      let rejectedCount = 0;

      if (promises.length === 0) {
        reject(new AggregateError([], 'All promises were rejected'));
        return;
      }

      promises.forEach((promise, index) => {
        MyPromise.resolve(promise).then(
          value => resolve(value),
          reason => {
            errors[index] = reason;
            rejectedCount++;
            if (rejectedCount === promises.length) {
              reject(new AggregateError(errors, 'All promises were rejected'));
            }
          }
        );
      });
    });
  }
}
```

## 实战案例

### 案例一：测试手写 Promise 的链式调用

```javascript
const p = new MyPromise((resolve) => {
  resolve(1);
});

p.then(value => {
  console.log(value);  // 1
  return value * 2;
}).then(value => {
  console.log(value);  // 2
  return value * 2;
}).then(value => {
  console.log(value);  // 4
});
```

### 案例二：值穿透测试

```javascript
MyPromise.resolve('直接值').then().then(value => {
  console.log(value);  // '直接值' —— 值穿透
});
// 没有传回调时，默认使用 value => value
```

### 案例三：thenable 测试

```javascript
const thenable = {
  then(resolve) {
    resolve('来自 thenable');
  }
};

MyPromise.resolve(thenable).then(value => {
  console.log(value);  // '来自 thenable'
});
// resolve 一个 thenable 对象时，会调用它的 then 方法
```

### 案例四：all 和 race 使用

```javascript
const p1 = new MyPromise(resolve => setTimeout(() => resolve(1), 100));
const p2 = new MyPromise(resolve => setTimeout(() => resolve(2), 200));
const p3 = new MyPromise((_, reject) => setTimeout(() => reject('err'), 50));

MyPromise.all([p1, p2]).then(values => {
  console.log(values);  // 100ms 后: [1, 2]
});

MyPromise.race([p1, p3])
  .then(v => console.log('胜出:', v))
  .catch(e => console.error('胜出:', e));  // 50ms 后: 'err'（p3 先 reject）
```

## 底层原理

### then 回调的异步执行

Promise/A+ 规范要求 then 的回调**必须在当前执行上下文结束后才能被调用**（异步）。我们使用 `queueMicrotask` 来模拟：

```javascript
// 浏览器中的微任务调度方法对比
// 1. queueMicrotask —— 推荐（现代浏览器）
// 2. MutationObserver —— 老式 polyfill
// 3. process.nextTick —— Node.js 环境
// 4. setTimeout(fn, 0) —— 回退方案（但这是宏任务）

// 如果用 setTimeout，会导致微任务/宏任务的执行顺序错误：
console.log('1');

new Promise(resolve => {
  resolve('Promise');
}).then(value => console.log(value));

console.log('3');

// 正确输出：1, 3, 'Promise'
// 如果用 setTimeout 模拟，输出会变成：1, 'Promise', 3 ❌
// 所以微任务模拟不能直接用 setTimeout
```

### Promise 执行流程详解

```javascript
new MyPromise((resolve) => {
  console.log('1');            // 同步执行
  resolve('2');
  console.log('3');            // 同步执行
}).then(value => {
  console.log(value);          // 异步执行（微任务）
});

console.log('4');

// 输出顺序：1, 3, 4, 2
// 
// 执行流程：
// 1. 进入构造函数，执行 executor
//    - 输出 '1'
//    - resolve('2') → 状态变为 fulfilled，value = '2'
//    - 输出 '3'
//    
// 2. then 方法内部，状态已是 fulfilled
//    - 将回调放入微任务队列（queueMicrotask）
//
// 3. 继续执行同步代码
//    - 输出 '4'
//
// 4. 同步代码执行完毕，执行微任务队列
//    - 执行 then 的回调：输出 '2'
```

## 高频面试题解析

### Q1：手写 Promise 时，then 方法为什么能链式调用？

**参考答案：**
因为 `then` 方法返回了一个**新的 Promise**。规范要求 `then` 必须返回一个新的 Promise，而不是 `this`（当前 Promise）。这样可以实现如下特性：

```javascript
promise.then(fn1).then(fn2).then(fn3)
// 每次 then 都产生新的 Promise，互不影响
// fn1 的返回值传给 fn2，fn2 的返回值传给 fn3
```

### Q2：如何处理 then 回调的返回值？

**参考答案：**
根据 Promise/A+ 规范，then 回调的返回值（记作 x）根据类型不同有不同的处理：
1. x 是 Promise 实例 → 等待 x 完成，其结果作为 promise2 的结果
2. x 是 thenable 对象 → 调用它的 then 方法
3. x 是普通值 → 直接 resolve(x)
4. 如果抛出异常 → reject(异常)
5. 如果 x === promise2 → throw TypeError（循环引用）

### Q3：手写 Promise 时，微任务怎么模拟？

**参考答案：**
优先使用 `queueMicrotask(fn)`（浏览器和 Node.js 12+ 都支持）。
降级方案：
- Node.js：`process.nextTick(fn)`
- 浏览器：`MutationObserver` + `document.createTextNode`
- 最差方案：`setTimeout(fn, 0)`（但这是宏任务，会破坏微任务语义）

### Q4：Promise.all 和 Promise.race 的实现要点？

**参考答案：**
**Promise.all：**
1. 返回一个新 Promise
2. 所有 promise 都成功 → resolve(结果数组)
3. 任何一个失败 → reject
4. 结果数组保持输入顺序
5. 空数组 → resolve([])

**Promise.race：**
1. 返回一个新 Promise
2. 第一个 promise 的状态决定整个 race 的结果
3. 不论成功还是失败，谁先到就是谁

### Q5：写一个完整的手写 Promise，主要步骤？

**参考答案：**
1. 定义三个状态常量
2. 创建构造函数，初始化状态和值
3. 实现 resolve/reject（状态不可逆）
4. 实现 then 方法（返回新 Promise）
5. resolvePromise 处理回调返回值
6. 处理异步场景（存储回调，状态改变后执行）
7. 实现 catch/finally
8. 实现静态方法 resolve/reject/all/race

## 总结与扩展

### 核心要点

| 概念 | 要点 |
|------|------|
| **状态机** | Pending → Fulfilled / Rejected，不可逆 |
| **then** | 返回新 Promise，支持链式调用 |
| **resolvePromise** | 处理回调返回值（值/Promise/thenable） |
| **异步执行** | 回调是微任务，用 queueMicrotask |
| **值穿透** | then 的回调为空时，默认 `v => v` 和 `r => { throw r }` |
