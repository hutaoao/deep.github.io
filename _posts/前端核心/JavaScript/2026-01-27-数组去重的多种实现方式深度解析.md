## 一句话概括

数组去重就是从数组中移除重复元素，核心是**判断两个元素是否"相等"**。JavaScript 提供了多种实现路径——从 ES6 一行代码的 `[...new Set(arr)]` 到支持自定义比较函数的复杂实现，每种方法在时间复杂度、空间复杂度和适用场景上存在显著差异。

## 背景与意义

### 为什么去重是"简单但不简单"的题？

去重看起来很简单——不就是把重复的去掉吗？但实际上：

```javascript
// ❌ 3 个不同的"相等"概念在打架
[1, '1']                    // 值不同
[null, undefined]           // 不同类型
[NaN, NaN]                  // NaN !== NaN
[{a:1}, {a:1}]              // 对象引用不同

// 去重涉及 JavaScript 最核心的语言特性：
// 类型系统、相等比较、哈希机制、引用 vs 值
```

### 面试高频信号

数组去重是**手写题必考**，问法有层次：

```
第 1 层（送分）："写出数组去重的常见方法"
  → 答出 Set / indexOf 就够

  → 考察性能意识
```

## 概念与定义

### 去重 = 过滤重复元素的算法

```javascript
// 去重的输入输出
const input    = [1, 2, 2, 3, 4, 4, 5, 1];
const expected = [1, 2, 3, 4, 5];
// 元素 2 和 4 各重复了一次，1 也重复了一次
```

### 判等逻辑的选择

| 判等方式 | 运算符 | 示例 | 去重效果 |
|---------|-------|------|---------|
| **严格相等** | `===` | `1 === 1` → true | 基本类型去重基础 |
| **SameValueZero** | Set/Map 内部 | `NaN === NaN` → true | Set 能去重 NaN 但 `===` 不行 |
| **SameValue** | `Object.is()` | `-0 !== 0` | 区分 -0 和 0 |
| **引用相等** | 对象对比 | `{a:1} === {a:1}` → false | 对象即使内容相同也判不同 |
| **深相等** | lodash `isEqual` | `{a:1} == {a:1}` → true | 按内容判等 |

## 最小示例

```javascript
// ✅ 日常最推荐：一行代码
const arr = [1, 2, 2, 3, 4, 4, 5, 1];
const unique = [...new Set(arr)];
console.log(unique);  // [1, 2, 3, 4, 5]

// ✅ 保留原顺序 + 基本类型去重
const result = arr.filter((item, index) => arr.indexOf(item) === index);
console.log(result);  // [1, 2, 3, 4, 5]

// ✅ 对象数组去重（按 key）
const users = [
  { id: 1, name: '张三' },
  { id: 2, name: '李四' },
  { id: 1, name: '张三' }  // 重复
];
const seen = new Set();
const uniqueUsers = users.filter(u => {
  const dup = seen.has(u.id);
  seen.add(u.id);
  return !dup;
});
// [{ id: 1, name: '张三' }, { id: 2, name: '李四' }]
```

## 核心知识点拆解

### 知识点 1：Set 去重详解

```javascript
const arr = [1, 2, 2, 3, 4, 4, 5, NaN, NaN, '1', '1'];

// Set 方法（推荐）
const unique = [...new Set(arr)];
console.log(unique);
// [1, 2, 3, 4, 5, NaN, '1']
// ★ Set 使用 SameValueZero 算法
// ★ NaN 被正确去重（NaN === NaN 在 Set 中被视为相等）
// ★ '1' 和 1 被视为不同（字符串和数字区分）
```

**Set 去重的优势**：
- 时间复杂度 O(n)，使用哈希表做查重
- 代码最简洁
- 保持插入顺序
- 正确处理 NaN

**Set 去重的局限**：
- ❌ 对象去重：`new Set([{a:1}, {a:1}])` → 长度 2（对象引用不同）
- ❌ 无法自定义比较逻辑（未来也许有 `collection.groupBy`）
- ❌ 类型转换不会自动发生

### 知识点 2：indexOf / includes 去重详解

```javascript
const arr = [1, 2, 2, 3, 4, 4, 5, NaN, '1'];

// indexOf 方案
function uniqueByIndexOf(arr) {
  return arr.filter((item, index) => arr.indexOf(item) === index);
  // 原理：如果当前元素第一次出现的位置就是当前位置，则保留
}
console.log(uniqueByIndexOf(arr));
// [1, 2, 3, 4, 5, '1']
// ★ NaN 被过滤掉了！因为 indexOf 使用严格相等比较
// arr.indexOf(NaN) 始终返回 -1（找不到）

// includes 方案
function uniqueByIncludes(arr) {
  const result = [];
  for (const item of arr) {
    if (!result.includes(item)) {
      result.push(item);
    }
  }
  return result;
}
console.log(uniqueByIncludes(arr));
// [1, 2, 3, 4, 5, NaN, '1']
// ★ includes 使用 SameValueZero，能正确处理 NaN
```

**性能对比**（10 万条数据）：

| 方法 | 时间复杂度 | 实测时间（10万条，约有100个唯一值） |
|------|:----------:|:--------------------------------:|
| `Set` | O(n) | ~5ms |
| `indexOf + filter` | O(n²) | ~3000ms |
| `includes + 循环` | O(n²) | ~3200ms |

**🎯 面试关键**：indexOf 去重性能差的根本原因是**每次 filter 回调都要遍历一次数组**。外层遍历 n 次，内层 indexOf 也遍历 n 次 → O(n²)。

### 知识点 3：Map / 对象映射去重

```javascript
// Map 去重（推荐用于混合类型）
function uniqueByMap(arr) {
  const map = new Map();
  return arr.filter(item => {
    if (map.has(item)) return false;
    map.set(item, true);
    return true;
  });
  // 性能 O(n)，且能处理各种类型
}

// 对象映射去重（只适用于基本类型）
function uniqueByObject(arr) {
  const obj = {};
  return arr.filter(item => {
    const key = String(item);  // 隐式转字符串
    if (obj[key]) return false;
    obj[key] = true;
    return true;
  });
}
console.log(uniqueByObject([1, '1', 2, '2']));
// [1] 或 ['1'] 中的一个 —— 问题！
// 因为 String(1) === '1'，数字 1 和字符串 '1' 被当作相同
```

**对象映射的陷阱**：

```javascript
const arr = [1, '1', undefined, null, NaN, {a:1}];

// 对象属性作为 map 的问题：
// 1. 键只能是 string 或 Symbol
// 2. '1' 和 1 被当作同一个键
// 3. null 和 undefined 被转为 'null' 和 'undefined'
// 4. 所有对象被转为 '[object Object]'

const obj = {};
arr.forEach(item => {
  obj[item] = true;  // 隐式类型转换
});
console.log(Object.keys(obj));
// ['1', 'undefined', 'null', 'NaN', '[object Object]']
// 丢失了重要信息！
```

### 知识点 4：对象数组去重

**最常被问到的场景**：

```javascript
const users = [
  { id: 1, name: '张三', age: 25 },
  { id: 2, name: '李四', age: 30 },
  { id: 1, name: '张三', age: 25 },  // 完全重复
  { id: 3, name: '王五', age: 28 },
  { id: 2, name: '李四', age: 31 },  // id 重复但 age 不同
];

// 方案 1：按特定 key 去重（最常用）
function uniqueByKey(arr, key) {
  const seen = new Set();
  return arr.filter(item => {
    const value = item[key];
    const isDuplicate = seen.has(value);
    seen.add(value);
    return !isDuplicate;
  });
}

console.log(uniqueByKey(users, 'id'));
// [{ id: 1, name: '张三', age: 25 },
//  { id: 2, name: '李四', age: 30 },
//  { id: 3, name: '王五', age: 28 }]
```

```javascript
// 方案 2：按多 key 组合去重
function uniqueByKeys(arr, keys) {
  const seen = new Set();
  return arr.filter(item => {
    const key = keys.map(k => item[k]).join('|');
    const isDuplicate = seen.has(key);
    seen.add(key);
    return !isDuplicate;
  });
}

// 方案 3：深比较去重（用 JSON.stringify）
function uniqueByDeepCompare(arr) {
  const seen = new Set();
  return arr.filter(item => {
    const key = JSON.stringify(item);
    const isDuplicate = seen.has(key);
    seen.add(key);
    return !isDuplicate;
  });
}
// 注意：JSON.stringify 对带有 undefined/函数/循环引用的对象会出问题
```

```javascript
// 方案 4：自定义比较器
function uniqueWithComparator(arr, comparator) {
  return arr.filter((item, index) =>
    arr.findIndex(other => comparator(item, other)) === index
  );
}

// 使用：按 id 去重
const r1 = uniqueWithComparator(users, (a, b) => a.id === b.id);
// 使用：按 name + age 去重
const r2 = uniqueWithComparator(users, (a, b) =>
  a.name === b.name && a.age === b.age
);
```

### 知识点 5：NaN、-0、null、undefined 等边界值处理

```javascript
const arr = [NaN, NaN, -0, 0, null, undefined, '', false];

// Set 的表现
console.log([...new Set(arr)]);
// [NaN, -0, 0, null, undefined, '', false]
// ★ NaN 被去重（Set 使用 SameValueZero）
// ★ -0 和 0 被视为相等（SameValueZero 认为它们相等）

// Object.is 区分 -0 和 0
console.log(Object.is(-0, 0));  // false

// 如果需要严格区分 -0：
function uniqueStrict(arr) {
  const seen = new Set();
  return arr.filter(item => {
    // 使用 Object.is 做严格的去重
    const found = [...seen].some(existing => Object.is(existing, item));
    if (!found) {
      seen.add(item);
      return true;
    }
    return false;
  });
}

console.log(uniqueStrict([-0, 0, 0, -0]));
// [-0, 0] —— 同时保留了 -0 和 0
```

## 实战案例

### 案例一：API 数据清洗

```javascript
// 实际项目：接口返回了重复的用户列表（分页兜底）
async function fetchAllUsers() {
  const allUsers = [];
  let page = 1;
  
  while (true) {
    const { data } = await api.get('/users', { page });
    if (data.length === 0) break;
    allUsers.push(...data);
    page++;
  }
  
  // ★ 跨页可能有重复用户，按 id 去重
  const seen = new Set();
  const uniqueUsers = [];
  
  for (const user of allUsers) {
    if (!seen.has(user.id)) {
      seen.add(user.id);
      uniqueUsers.push(user);
    }
  }
  
  return uniqueUsers;
}
```

### 案例二：前端数组操作注意事项

```javascript
// ★ 删除数组中的重复元素（就地修改）
function removeDuplicatesInPlace(arr) {
  // 先用 Set 获取唯一值
  const seen = new Set();
  let writeIndex = 0;
  
  for (const item of arr) {
    if (!seen.has(item)) {
      seen.add(item);
      arr[writeIndex] = item;
      writeIndex++;
    }
  }
  
  // 截断多余的元素
  arr.length = writeIndex;
  return arr;
}

// ★ 数组去重后保持原始顺序（保留首次出现）
function uniquePreserveOrder(arr) {
  const seen = new Map();
  return arr.filter((item, index) => {
    if (seen.has(item)) return false;
    seen.set(item, index);  // 记录首次出现位置
    return true;
  });
}
```

## 底层原理

### Set 与 Map 的哈希表实现

JavaScript 引擎（V8）对 Set 和 Map 使用类似哈希表的实现：

```
Set 内部结构（简化）：

当执行 new Set([1, 2, 2, 3])：

1. 创建哈希表（初始容量 4 或 8）
2. 插入 1：
   计算 hash(1) → 索引 0
   table[0] = { key: 1, next: null }

3. 插入 2：
   计算 hash(2) → 索引 2
   table[2] = { key: 2, next: null }

4. 插入第二个 2：
   计算 hash(2) → 索引 2
   检查 table[2] 的 key === 2 → 已存在，不插入

5. 插入 3：类似

最终存储：{1, 2, 3}
```

**Hash 冲突处理**：使用链地址法（Separate Chaining），当冲突发生时，在同索引位置用链表存储。

**动态扩容**：当元素数量超过负载因子（约 75%）时，自动扩容并 rehash。

### 各种去重方法的 V8 优化级别

```javascript
// 1. Set 方法 → V8 高度优化（内置哈希表实现）
const r1 = [...new Set(arr)];
// V8 为 Set 原生实现了快速哈希查找

// 2. Map 方法 → V8 优化（内置实体的 SameValueZero 比较）
const r2 = arr.filter((() => { const m = new Map(); return v => m.has(v) ? false : (m.set(v, 1), true); })());

// 3. indexOf 方法 → 无优化，线性查找
const r3 = arr.filter((v, i) => arr.indexOf(v) === i);
// V8 无法优化这种 O(n²) 的扫描模式

// 4. 对象属性法 → V8 有 HiddenClass 优化
const obj = {};
const r4 = arr.filter(v => obj[v] ? false : (obj[v] = true));
// 但存在类型转换问题
```

## 高频面试题解析

### Q1：数组去重有哪些实现方式？最推荐哪种？

**参考答案：**
主要实现方式：
1. **Set 方法** `[...new Set(arr)]`——O(n)，最推荐，一行搞定
2. **filter + indexOf** —— O(n²)，适合面试展示思路
3. **Map 方法** —— O(n)，适合混合类型
4. **对象映射法** —— O(n)，但存在类型转换问题

**最推荐**：基本类型用 Set，对象数组按 key 用 Map/Set，需要自定义比较用 filter + findIndex。

### Q2：Set 方法去重的原理是什么？为什么能正确处理 NaN？

**参考答案：**
Set 内部使用 **SameValueZero** 比较算法，而不是 `===`。SameValueZero 将 `NaN` 视为等于自身（而 `NaN === NaN` 是 false）。所以 Set 能正确去重 NaN。其他区别：SameValueZero 将 `-0` 和 `0` 视为相等（而 Object.is 区分它们）。

### Q3：对象数组如何去重？

**参考答案：**
对象不能直接用 Set 去重（引用不同）。
1. **按 key**：`Set` + `filter`，根据 id 等唯一标识判断
2. **按多 key**：用 `|` 组合多个 key 作为 key
3. **深比较**：用 `JSON.stringify` 转为字符串比较（注意局限性）
4. **自定义**：传入比较器函数做更灵活的比较

### Q4：如果数组有 10 万条数据，怎么选择去重方法？

**参考答案：**
- Set/Map：O(n)，首选，10 万条仅需 ~5ms
- 避免 O(n²)：indexOf/filter/includes
- 超大数组（百万级）：考虑分批处理或 Web Worker
