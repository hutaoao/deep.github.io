## 一句话概括

原型（Prototype）是 JavaScript 实现**对象间属性共享**的机制——每个对象都有一个隐藏的 `[[Prototype]]` 引用指向它的"模板对象"（原型），访问属性时先查自身，没找到就沿原型链向上查找直到 `null`。原型链是 JS 实现"继承"的唯一途径。

## 背景与意义

### 为什么需要原型？

在 JavaScript 诞生之初（1995年，10天内设计完成），Brendan Eich 需要实现"对象继承"的机制，但不想引入 Java 式的类（class）体系。

```javascript
// ❌ 如果没有原型，每个实例都复制一份方法
function createUser(name) {
  return {
    name: name,
    sayHi: new Function('console.log("Hi, I\'m " + this.name)'),  // 每次创建一个新函数
    // 1000 个用户 → 1000 个 sayHi 函数 → 内存浪费
  };
}

// ✅ 有了原型，所有实例共享方法
function User(name) {
  this.name = name;
}
User.prototype.sayHi = function() {
  console.log("Hi, I'm " + this.name);
};
// 1000 个用户 → 1 个 sayHi 函数 → 内存复用
```

**原型解决了核心问题：内存复用 + 动态继承**。

## 概念与定义

### 两个核心属性

```javascript
// 1. prototype（函数特有属性）
// 函数都有一个 prototype 属性，指向"实例的原型对象"
function Person(name) {
  this.name = name;
}
Person.prototype.sayHi = function() {
  console.log(`Hi, I'm ${this.name}`);
};

// 2. __proto__（所有对象都有）
// 指向构造函数的 prototype
const p = new Person('Alice');
console.log(p.__proto__ === Person.prototype);  // true
```

```
对象关系图：

Person 函数
  │
  ├── Person.prototype ────────→ { sayHi: fn, constructor: Person }
  │                                     ↑
  │                                     │
  p = new Person('Alice') ──→ p.__proto__─┘
  │
  p 的自身属性：{ name: 'Alice' }
  p 的原型属性：通过 __proto__ 访问 Person.prototype
```

### 原型链的查找规则

```javascript
const arr = [1, 2, 3];

// 访问 arr.length
arr.length  // 自身属性，直接找到

// 访问 arr.map
arr.map     // 自身没有 → 去 arr.__proto__ (Array.prototype) 找 → 找到

// 访问 arr.toString
arr.toString  // 自身没有 → Array.prototype 没有 → 去 Array.prototype.__proto__ (Object.prototype) 找 → 找到

// 访问 arr.nonexist
arr.nonexist  // 自身没有 → Array.prototype 没有 → Object.prototype 没有 → null → undefined
```

```
原型链：
arr → Array.prototype → Object.prototype → null
      ↑               ↑                    ↑
    Array.map    Object.toString       原型链尽头
```

## 最小示例

```javascript
// 构造函数
function Animal(type) {
  this.type = type;
}

Animal.prototype.eat = function() {
  console.log(`${this.type} is eating`);
};

// 实例
const dog = new Animal('dog');
console.log(dog.type);      // 'dog'（自身属性）
dog.eat();                   // 'dog is eating'（原型上的方法）

// 原型链
console.log(dog.__proto__ === Animal.prototype);                // true
console.log(Animal.prototype.__proto__ === Object.prototype);   // true
console.log(Object.prototype.__proto__);                       // null（尽头）

// dog 的原型链：dog → Animal.prototype → Object.prototype → null
```

## 核心知识点拆解

### 知识点 1：`__proto__` vs `prototype` 的区别

**这是面试中 90% 的人会搞混的问题。**

```javascript
// ★ prototype 只在函数上有
function Foo() {}
console.log(typeof Foo.prototype);  // 'object'——函数都有 prototype

// ★ __proto__ 在对象上有（包括函数对象！）
const obj = {};
console.log(typeof obj.__proto__);  // 'object'

// ★ 函数既是函数也是对象
console.log(Foo.__proto__ === Function.prototype);  // true
console.log(Foo.prototype.__proto__ === Object.prototype);  // true
```

| 属性 | 拥有者 | 用途 | 例子 |
|------|-------|------|------|
| `prototype` | **函数** | 指向构造函数的"模板"，new 出来的实例会继承它 | `function Foo() {}.prototype` |
| `__proto__` | **所有对象**（包括函数） | 指向创建它的构造函数的 prototype | `obj.__proto__` |
| `[[Prototype]]` | **所有对象**（引擎内部） | `__proto__` 的实际名称，但标准推荐用 `Object.getPrototypeOf` | `Object.getPrototypeOf(obj)` |

```javascript
// ★ 特殊关系：所有函数的 __proto__ 都等于 Function.prototype
function foo() {}
const arrow = () => {};

console.log(foo.__proto__ === Function.prototype);    // true
console.log(arrow.__proto__ === Function.prototype);  // true
console.log(Function.prototype.__proto__ === Object.prototype);  // true

// ★ Object 本身也是函数
console.log(Object.__proto__ === Function.prototype);  // true

// ★ 非常重要：constructor 属性
// prototype 上默认有一个 constructor，指回构造函数
function Func() {}
console.log(Func.prototype.constructor === Func);  // true

const f = new Func();
console.log(f.constructor === Func);  // true（通过原型链找到的）
```

### 知识点 2：new 操作符的手写实现

```javascript
function myNew(constructor, ...args) {
  // 1. 创建空对象，__proto__ 指向 constructor.prototype
  const obj = Object.create(constructor.prototype);
  
  // 2. 绑定 this，执行构造函数
  const result = constructor.apply(obj, args);
  
  // 3. 如果构造函数返回对象，则返回那个对象；否则返回新对象
  return result instanceof Object ? result : obj;
}

// 使用
function Person(name, age) {
  this.name = name;
  this.age = age;
}

Person.prototype.sayHi = function() {
  console.log(`Hi, I'm ${this.name}`);
};

const p = myNew(Person, 'Alice', 25);
p.sayHi();  // "Hi, I'm Alice"
console.log(p.__proto__ === Person.prototype);  // true
```

### 知识点 3：原型链继承的演进

**这是原型链面试中最重要的笔试/手写题。**

```javascript
// ===== 版本 1：原型链继承 =====
function Parent() {
  this.name = 'parent';
  this.colors = ['red', 'blue'];  // 引用类型！
}
Parent.prototype.sayHi = function() {
  console.log('Hi from ' + this.name);
};

function Child() {}
Child.prototype = new Parent();

const c1 = new Child();
const c2 = new Child();

c1.colors.push('green');
console.log(c2.colors);  // ['red', 'blue', 'green'] ← 被 c1 影响了！
// 问题：引用类型被所有实例共享
```

```javascript
// ===== 版本 2：借用构造函数（经典继承） =====
function Parent(name) {
  this.name = name;
  this.colors = ['red', 'blue'];
}

function Child(name) {
  Parent.call(this, name);  // 借用构造函数
}

const c1 = new Child('c1');
const c2 = new Child('c2');

c1.colors.push('green');
console.log(c2.colors);  // ['red', 'blue'] ← 不受影响 ✅

// 问题：方法无法复用（每次创建新函数）
// 如果 Parent.prototype 上有方法，Child 的实例访问不到
```

```javascript
// ===== 版本 3：组合继承（常用） =====
function Parent(name) {
  this.name = name;
  this.colors = ['red', 'blue'];
}
Parent.prototype.sayHi = function() {
  console.log('Hi from ' + this.name);
};

function Child(name, age) {
  Parent.call(this, name);  // 第一次调用 Parent
  this.age = age;
}

Child.prototype = new Parent();  // 第二次调用 Parent
Child.prototype.constructor = Child;  // 修复 constructor
Child.prototype.sayAge = function() {
  console.log(this.age);
};

const c1 = new Child('c1', 10);
c1.colors.push('green');
console.log(c1.colors);  // ['red', 'blue', 'green'] ✅
c1.sayHi();  // "Hi from c1" ✅

// 问题：Parent 被调用了两次，不必要的内存浪费
```

```javascript
// ===== 版本 4：寄生组合继承（完美方案） =====
function inheritPrototype(Child, Parent) {
  // 用 Object.create 创建一个"纯净"的原型
  Child.prototype = Object.create(Parent.prototype);
  // 修复 constructor 引用
  Child.prototype.constructor = Child;
}

function Parent(name) {
  this.name = name;
  this.colors = ['red', 'blue'];
}
Parent.prototype.sayHi = function() {
  console.log('Hi from ' + this.name);
};

function Child(name, age) {
  Parent.call(this, name);  // 只调用一次 Parent
  this.age = age;
}

inheritPrototype(Child, Parent);
Child.prototype.sayAge = function() {
  console.log(this.age);
};

const c1 = new Child('c1', 10);
// ✅ 只调用了一次 Parent
// ✅ 属性不共享
// ✅ 方法可复用
// ✅ 原型链完整
```

### 知识点 4：class 语法与原型的关系

ES6 的 `class` 本质是原型继承的**语法糖**：

```javascript
class Animal {
  constructor(type) {
    this.type = type;
  }
  
  eat() {
    console.log(`${this.type} is eating`);
  }
  
  static create(type) {
    return new Animal(type);
  }
}

class Dog extends Animal {
  constructor(name) {
    super('dog');
    this.name = name;
  }
  
  bark() {
    console.log(`${this.name} is barking`);
  }
}

// 等价于：
function Animal(type) {
  this.type = type;
}
Animal.prototype.eat = function() { ... };
Animal.create = function(type) { ... };  // 静态方法

// extends 等价于原型链继承：
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;
```

## 实战案例

### 案例一：属性屏蔽（Property Shadowing）

```javascript
function Parent() {}
Parent.prototype.value = 42;

function Child() {}
Child.prototype = Object.create(Parent.prototype);

const obj = new Child();

// 读取
console.log(obj.value);  // 42（原型链上的）

// 写入
obj.value = 100;  // 在 obj 自身创建新属性，不会修改原型
console.log(obj.value);         // 100（自身属性屏蔽了原型）
console.log(obj.__proto__.value);  // 42（原型上的没变）

delete obj.value;
console.log(obj.value);  // 42（删掉自身属性后，又从原型链找到）
```

### 案例二：instanceof 原理

```javascript
// instanceof 检查右侧构造函数的 prototype 是否在左侧对象的原型链上
function myInstanceof(left, right) {
  let proto = Object.getPrototypeOf(left);
  
  while (true) {
    if (proto === null) return false;
    if (proto === right.prototype) return true;
    proto = Object.getPrototypeOf(proto);
  }
}

const arr = [];
console.log(myInstanceof(arr, Array));      // true
console.log(myInstanceof(arr, Object));     // true
console.log(myInstanceof(arr, RegExp));     // false
```

### 案例三：原型链与类型检测

```javascript
const arr = [1, 2, 3];

// typeof —— 无法区分对象类型
console.log(typeof arr);   // 'object'
console.log(typeof {});    // 'object'

// instanceof —— 检查原型链
console.log(arr instanceof Array);   // true
console.log(arr instanceof Object);  // true（Array 继承自 Object）

// Object.prototype.toString —— 最准确
const toString = Object.prototype.toString;
console.log(toString.call(arr));         // '[object Array]'
console.log(toString.call({}));          // '[object Object]'
console.log(toString.call(new Date()));  // '[object Date]'

// Array.isArray —— 数组专用
console.log(Array.isArray(arr));  // true
```

## 底层原理

### V8 引擎对原型链的优化

V8 使用 **Hidden Class** 和 **Inline Cache（IC）** 加速原型链查找：

```
原型链属性访问的 V8 优化过程：

1. 第一次访问 obj.sayHi：
   引擎在 obj 自身找 → 没有
   在 obj.__proto__（Parent.prototype）找 → 找到
   建立 IC：下次直接从 Parent.prototype 取

2. 第二次访问 obj.sayHi：
   检查 Hidden Class 是否变化 → 没有 → 直接使用缓存的地址
   跳过原型链查找，跳转到 sayHi 的代码

★ 这就是原型动态性的代价：
   如果在运行时修改了原型（如 obj.__proto__ = something），
   V8 需要废弃 IC 缓存，重新编译，严重影响性能
```

**原因**：V8 假设原型链是稳定的。如果经常修改 `__proto__`，会导致所有函数的 IC 缓存失效。

## 高频面试题解析

### Q1：`__proto__` 和 `prototype` 的区别？

**参考答案：**
- `prototype` 是**函数**的属性，指向函数的原型对象。当函数作为构造函数被 `new` 调用时，新创建对象的 `[[Prototype]]` 会指向它
- `__proto__` 是**任何对象**的属性（包括函数），指向创建该对象的构造函数的 `prototype`
- 推荐用 `Object.getPrototypeOf(obj)` 代替 `obj.__proto__`

### Q2：new 操作符的执行过程是什么？

**参考答案：**
1. 创建一个空对象
2. 将空对象的 `[[Prototype]]` 指向构造函数的 `prototype`（`Object.create(Constructor.prototype)`）
3. 将构造函数的 `this` 绑定到新对象，执行构造函数代码
4. 如果构造函数返回了对象，返回该对象；否则返回新创建的对象

### Q3：原型链继承的几种方式及优缺点？

**参考答案：**
1. **原型链继承**：`Child.prototype = new Parent()` —— 引用类型被共享
2. **借用构造函数**：`Parent.call(this)` —— 方法不可复用
3. **组合继承**：两者结合 —— Parent 被调用两次
4. **寄生组合继承**：`Object.create(Parent.prototype)` —— 最完美

### Q4：原型链的尽头是什么？

**参考答案：**
`null`。`Object.prototype.__proto__ === null`，所有对象的原型链最终都指向 `null`。

### Q5：`function Foo() {}` 的 Foo.prototype.constructor 指向谁？

**参考答案：**
指向 `Foo`。函数的 `prototype` 对象默认有一个 `constructor` 属性，指回函数本身。

## 总结与扩展

### 核心要点

| 概念 | 说明 |
|------|------|
| **`prototype`** | 函数特有，指向实例的"模板" |
| **`__proto__`** | 所有对象都有，指向创建它的构造函数的 prototype |
| **原型链** | 属性查找路径：自身 → `__proto__` → ... → `null` |
| **new** | 创建对象 + 链接原型 + 绑定 this + 返回对象 |
| **instanceof** | 检查 prototype 是否在原型链上 |
