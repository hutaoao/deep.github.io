> 一句话概括：条件类型是 TypeScript 类型系统中最强大的能力之一，它让类型也可以像函数一样"根据输入返回不同的输出"，实现了真正的类型级编程。

## 背景

TypeScript 的类型系统从早期的"静态类型标注"演进到今天的高度灵活的"类型编程"时代，条件类型（Conditional Types）是这个演进过程中的标志性特性。有了条件类型，我们可以写出"根据类型 A 是否可赋值给类型 B 来决定返回类型 C 还是类型 D"这样的类型逻辑——这已经超越了传统静态类型的范畴。

## 概念与定义

### 条件类型的基本语法

```typescript
T extends U ? X : Y
```

解读：**如果类型 `T` 可赋值给类型 `U`，则条件类型的结果为类型 `X`，否则为类型 `Y`**。

```typescript
type IsString<T> = T extends string ? 'yes' : 'no';

type A = IsString<string>;   // 'yes'
type B = IsString<number>;   // 'no'
type C = IsString<'hello'>;  // 'yes'（字符串字面量也是 string）
```

### 分布条件类型（Distributive Conditional Types）

当条件类型的左侧是**裸类型参数**（即没有经过任何包装的泛型参数）时，TypeScript 会对联合类型进行**自动分发**：

```typescript
type ToArray<T> = T extends any ? T[] : never;

type Result = ToArray<string | number>;
// 实际上是：
// ToArray<string> | ToArray<number>
// = string[] | number[]
```

**这是条件类型最核心、最容易出错的特性。**

如果不想分发，可以用元组包装：

```typescript
type ToArrayNonDist<T> = [T] extends [any] ? T[] : never;

type Result = ToArrayNonDist<string | number>;
// 结果： (string | number)[]
```

## 核心知识点拆解

### 1. 内置条件类型

TypeScript 内置了几个极其常用的条件类型：

```typescript
// 排除 U 中可赋值给 T 的类型
type Exclude<T, U> = T extends U ? never : T;

// 保留 U 中可赋值给 T 的类型
type Extract<T, U> = T extends U ? T : never;

// 去除 null 和 undefined
type NonNullable<T> = T extends null | undefined ? never : T;

// 提取函数返回值类型
type ReturnType<T> = T extends (...args: any) => infer R ? R : any;

// 提取函数参数类型
type Parameters<T> = T extends (...args: infer P) => any ? P : never;
```

**手写 Exclude 和 Extract 原理：**

```typescript
// Exclude: 从 T 中排除可以赋值给 U 的类型
type MyExclude<T, U> = T extends U ? never : T;

// 'a' | 'b' | 'c' 中：
// 'a' extends 'a'/'b' → never
// 'b' extends 'a'/'b' → never
// 'c' extends 'a'/'b' → 'c'
type R = MyExclude<'a' | 'b' | 'c', 'a' | 'b'>;  // 'c'

// Extract: 从 T 中保留可以赋值给 U 的类型
type MyExtract<T, U> = T extends U ? T : never;

type R2 = MyExtract<'a' | 'b' | 'c', 'a' | 'b'>;  // 'a' | 'b'
```

### 2. 条件类型的分发机制详解

```typescript
type Foo<T> = T extends 'a' | 'b' ? T : never;

// 分发过程：
// (T = 'a' | 'b' | 'c')
// 'a' extends 'a' | 'b' ? 'a' : never  → 'a'
// 'b' extends 'a' | 'b' ? 'b' : never  → 'b'
// 'c' extends 'a' | 'b' ? 'c' : never  → never
// 最终：'a' | 'b'

type R = Foo<'a' | 'b' | 'c'>;  // 'a' | 'b'
```

**触发分发的条件：类型参数本身是裸的泛型参数。** 以下情况不触发分发：

```typescript
type NoDist<T> = [T] extends [any] ? T[] : never;
// 元组包装后不触发分发
```

### 3. 在映射类型中使用条件类型

```typescript
// 将所有函数类型变为返回 void，不保留其他类型
type ToVoid<T> = {
  [K in keyof T]: T[K] extends (...args: any[]) => any 
    ? (...args: Parameters<T[K]>) => void 
    : T[K]
};

type R = ToVoid<{ 
  name: string; 
  greet(): string; 
  onClick(): void 
}>;
// { name: string; greet(): void; onClick(): void }
```

## 实战案例

### 案例：类型安全的 API 响应处理

```typescript
type ApiResponse<T, E = unknown> = 
  T extends { code: infer C; data: infer D } 
    ? C extends 200 
      ? { success: true; data: D } 
      : { success: false; error: E } 
    : { success: false; error: 'invalid response' };

// 使用
type R1 = ApiResponse<{ code: 200; data: { id: 1 } }>;
// { success: true; data: { id: 1 } }

type R2 = ApiResponse<{ code: 404; data: null }>;
// { success: false; error: unknown }
```

### 案例：递归条件类型（深度展开）

```typescript
// 深度展开数组类型
type DeepArray<T> = T extends (infer U)[]
  ? DeepArray<U>[] | DeepArray<U>
  : T;

type R = DeepArray<number[][]>;   // number
type R2 = DeepArray<string[][][]>; // string

// 实战：从联合类型中过滤掉 null/undefined
type NotNull<T> = T extends null | undefined ? never : T;

type R3 = NotNull<string | null | undefined | number>;
// string | number
```

### 案例：基于条件的工具类型

```typescript
// 根据属性值类型决定处理方式
type Stringify<T> = {
  [K in keyof T]: T[K] extends string | number
    ? string
    : T[K] extends boolean
      ? 'yes' | 'no'
      : T[K];
};

// 或者根据 key 名称做条件处理
type KeysToCamelCase<T> = {
  [K in keyof T as K extends `${infer P}_${infer Q}${infer R}` 
    ? `${P}${Uppercase<Q>}${R}` 
    : K]: T[K];
};

type snake_case = { user_name: string; created_at: number };
type camelCase = KeysToCamelCase<snake_case>;
// { userName: string; createdAt: number }
```

## 底层原理

### 条件类型的编译时求值

TypeScript 编译器在类型检查阶段对条件类型进行**短路求值**：

1. 解析 `T extends U ? X : Y` 语法树
2. 判断 `T` 是否可以赋值给 `U`（可赋值性检查）
3. 如果可以，结果替换为 `X`；否则替换为 `Y`
4. 对于联合类型的裸泛型，**递归对每个联合成员执行上述过程并取联合**

### 条件类型的延迟求值（Deferred Resolution）

当条件类型中的 `T` 仍然是泛型参数时，TypeScript 不会立即求值，而是创建**待定条件类型**。只有在泛型参数被具体类型填充时，才会真正求值：

```typescript
type Identity<T> = T extends any ? T : never;

// 此时 Identity<T> 仍然是待定状态，不进行分发
// 只有当传入具体类型时，才会展开：
type R = Identity<string | number>;  // 触发分发 → string | number
```

## 高频面试题解析

### Q1：什么是条件类型的分发机制？什么情况下不会触发分发？

**答：** 当条件类型左侧是裸泛型参数 `T` 时，如果 `T` 是联合类型，TypeScript 会对联合的每个成员分别求条件类型并取并集，这就是分发。不会触发分发的场景：① 用元组/数组包装：`[T] extends [any]`；② 对 `T` 做了映射处理后才参与条件判断；③ `T` 不是泛型参数而是具体类型。

### Q2：手写 `ReturnType` 并解释 `infer` 的作用。

**答：** `infer` 是 TypeScript 2.8 引入的关键字，用于在条件类型的 `extends` 子句中**声明一个局部类型变量**，后续可在 `true` 分支中引用它：

```typescript
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

// 语义：如果 T 能赋值给"参数任意、返回值为 R 的函数"，
// 则 R 就是我们要提取的返回值类型
type F = () => Promise<string>;
type R = ReturnType<F>;  // R = Promise<string>
```

### Q3：`Exclude<T, U>` 和 `Extract<T, U>` 的区别是什么？

**答：** 两者都是条件类型的应用，但逻辑相反：
- `Exclude<T, U>`：**从 T 中排除**能赋值给 U 的成员 → `T extends U ? never : T`
- `Extract<T, U>`：**从 T 中保留**能赋值给 U 的成员 → `T extends U ? T : never`

```typescript
type R = Exclude<'a' | 'b' | 'c', 'a'>;  // 'b' | 'c'
type R2 = Extract<'a' | 'b' | 'c', 'a'>;  // 'a'
```

## 总结与扩展

### 核心要点

1. 条件类型 `T extends U ? X : Y` 根据可赋值性返回不同类型
2. 裸泛型参数会触发联合类型的自动分发
3. `infer` 关键字用于从类型中"提取"子类型
4. 内置的 `Exclude`、`Extract`、`NonNullable`、`ReturnType` 都是条件类型的应用
5. 条件类型支持递归，可以构建深度类型操作
6. 分发行为可能导致意外的联合类型膨胀，要注意控制

### 实际工程建议

- 利用条件类型为第三方库打"补丁"，在不修改源码的情况下增强类型
- 善用 `Exclude` 和 `Extract` 做类型过滤，比手动写联合类型更安全
- 当遇到"这个类型随另一个类型变化"的场景时，优先考虑条件类型
- 递归条件类型要注意设置终止条件，防止无限递归导致 TS 服务器卡顿
