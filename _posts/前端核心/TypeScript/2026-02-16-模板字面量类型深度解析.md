## 一句话概括

**模板字面量类型（Template Literal Types）让 TypeScript 的类型系统具备了字符串级别的运算能力**：你可以像操作值一样在类型层面拼接、拆分、匹配字符串，这是高级类型体操的核心工具。

## 背景

TypeScript 4.1 引入了模板字面量类型，这是类型系统从"结构描述"走向"字符串计算"的重要里程碑。

面试中常见问题：
- "什么是模板字面量类型？它解决了什么问题？"
- "如何在类型层面做字符串拼接和匹配？"
- "用它实现一个类似于 `Uppercase` 的内置工具类型"

在实际开发中，模板字面量类型广泛用于：路由参数类型推断、CSS 属性类型约束、事件名称类型约束、国际化 key 类型推断等场景。

## 概念与定义

### 基础语法

模板字面量类型的语法与 JavaScript 模板字符串一致，但它作用于**类型空间**：

```typescript
// JavaScript 值空间：字符串拼接
const name = "TypeScript";
const greeting = `Hello, ${name}`;  // "Hello, TypeScript"

// TypeScript 类型空间：字符串字面量类型拼接
type Name = "TypeScript";
type Greeting = `Hello, ${Name}`;  // "Hello, TypeScript"
```

### 字符串联合类型的分布式拼接

当模板中的占位符是**联合类型**时，结果会自动分布式展开（笛卡尔积）：

```typescript
type Language = "TypeScript" | "JavaScript" | "Rust";
type Version = "v1" | "v2" | "v3";

// 自动生成 3 × 3 = 9 种组合
type Release = `${Language}-${Version}`;
// "TypeScript-v1" | "TypeScript-v2" | "TypeScript-v3"
// | "JavaScript-v1" | "JavaScript-v2" | "JavaScript-v3"
// | "Rust-v1" | "Rust-v2" | "Rust-v3"
```

## 最小示例

### 示例 1：事件名称类型约束

```typescript
type EventName = "click" | "focus" | "blur";
type HandlerName = `on${Capitalize<EventName>}`;
// "onClick" | "onFocus" | "onBlur"

function addEventListener(event: HandlerName, callback: () => void) {
  // ...
}

addEventListener("onClick", () => {});   // ✅
addEventListener("onFocus", () => {});   // ✅
// addEventListener("onHover", () => {}); // ❌ 类型错误
```

### 示例 2：CSS 单位约束

```typescript
type CSSUnit = "px" | "em" | "rem" | "%";
type CSSValue = `${number}${CSSUnit}`;

const width: CSSValue = "100px";   // ✅
const margin: CSSValue = "1.5rem"; // ✅
// const height: CSSValue = "100";  // ❌ 缺少单位
// const size: CSSValue = "100vh";  // ❌ vh 不在 CSSUnit 中
```

### 示例 3：路由参数推断

```typescript
type Route = `/user/${string}` | `/post/${number}`;

type UserRoute = Extract<Route, `/user/${string}`>;  // `/user/${string}`
type PostRoute = Extract<Route, `/post/${number}`>;  // `/post/${number}`
```

## 核心知识点拆解

### 1. 内置字符串操作类型

TypeScript 内置了四个用于模板字面量类型的工具类型：

```typescript
type UppercaseHello = Uppercase<"hello">;       // "HELLO"
type LowercaseHello = Lowercase<"HELLO">;       // "hello"
type CapitalizeHello = Capitalize<"hello">;     // "Hello"
type UncapitalizeHello = Uncapitalize<"Hello">; // "hello"
```

这些内置类型可以与模板字面量类型组合使用。

### 2. 联合类型的分布式特性

```typescript
type Action = "create" | "update" | "delete";
type Entity = "User" | "Post" | "Comment";

// 笛卡尔积展开
type ActionName = `${Capitalize<Action>}${Entity}`;
// "CreateUser" | "CreatePost" | "CreateComment"
// | "UpdateUser" | "UpdatePost" | "UpdateComment"
// | "DeleteUser" | "DeletePost" | "DeleteComment"
```

当联合类型非常大时（上百个成员），生成的类型可能会超出 TypeScript 编译器的限制（约 10 万个成员的阈值），需要注意性能。

### 3. infer + 模板字面量类型的模式匹配

这是模板字面量类型最强大的用法——在条件类型中使用 `infer` 进行字符串模式匹配：

```typescript
// 提取字符串中的数字
type ExtractNumber<T extends string> =
  T extends `${infer N extends number}` ? N : never;

type A = ExtractNumber<"123">;   // 123
type B = ExtractNumber<"abc">;   // never

// 提取路由参数
type RouteParams<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? { [K in Param]: string } & RouteParams<`/${Rest}`>
    : T extends `${string}:${infer Param}`
      ? { [K in Param]: string }
      : {};

type Params = RouteParams<"/user/:id/post/:postId">;
// { id: string; postId: string }
```

### 4. 递归模板字面量类型

TypeScript 4.5+ 支持递归条件类型中的模板字面量类型：

```typescript
// 字符串分割为联合类型
type Split<
  S extends string,
  Delimiter extends string
> = S extends `${infer Head}${Delimiter}${infer Tail}`
  ? Head | Split<Tail, Delimiter>
  : S;

type Words = Split<"a,b,c,d", ",">;
// "a" | "b" | "c" | "d"

// 字符串替换
type Replace<
  S extends string,
  From extends string,
  To extends string
> = S extends `${infer Head}${From}${infer Tail}`
  ? `${Head}${To}${Tail}`
  : S;

type Result = Replace<"Hello World", "World", "TypeScript">;
// "Hello TypeScript"
```

## 实战案例

### 案例 1：Redux Action 类型约束

```typescript
// 定义 action 类型
type Modules = "user" | "product" | "order";
type Actions = "FETCH" | "CREATE" | "UPDATE" | "DELETE";
type Status = "REQUEST" | "SUCCESS" | "FAILURE";

// 生成标准 Redux action 类型
type ReduxActionType = `${Uppercase<Actions>}_${Uppercase<Modules>}_${Status}`;

// 生成 3 × 4 × 3 = 36 个 action 类型
// "FETCH_USER_REQUEST" | "FETCH_USER_SUCCESS" | ...
```

### 案例 2：对象属性路径类型

```typescript
interface User {
  name: string;
  address: {
    city: string;
    street: string;
    zip: number;
  };
  hobbies: string[];
}

// 获取对象的所有属性路径（点分隔）
type Path<T, Prefix extends string = ""> = T extends object
  ? {
      [K in keyof T & string]: T[K] extends object
        ? Path<T[K], `${Prefix}${K}.`>
        : `${Prefix}${K}`;
    }[keyof T & string]
  : never;

type UserPaths = Path<User>;
// "name" | "address.city" | "address.street" | "address.zip" | "hobbies"
```

### 案例 3：国际化的类型安全 key

```typescript
// 定义翻译 key 结构
type Locale = "zh-CN" | "en-US" | "ja-JP";
type Section = "common" | "error" | "validation";
type Key = string;

// 生成类型安全的翻译 key
type I18nKey = `${Section}.${Key}`;

// 使用示例
function t(key: I18nKey, locale: Locale): string {
  // ...
}

t("common.save", "zh-CN");     // ✅
t("error.network", "en-US");   // ✅
// t("unknown.key", "zh-CN");  // ❌ "unknown" 不在 Section 中
```

## 实用技巧：模板字面量类型的注意事项

### 联合类型展开别太贪心

模板字面量类型中，联合类型占位符会产生**笛卡尔积展开**。多个大联合类型组合时，结果数量会爆炸性增长。

```typescript
// 安全范围：几 × 几 × 几，组合后几十到几百个成员
type Action = "create" | "update" | "delete";  // 3 个
type Entity = "User" | "Post";                   // 2 个
type Safe = `${Action}_${Entity}`;                // 6 个 ✅

// 危险范围：几十 × 几十，组合后上千甚至上万
// type AllTags = `$"{50个标签}_${50个状态}`  // 2500 个 ✅ 还行
// type TooMany = `$"{50个前缀}_${50个中缀}_${50个后缀}`  // 125,000 ❌ 超过编译器限制
```

🔥 **面试考点**："模板字面量类型产生太多联合成员会怎样？"——TypeScript 会报 `Expression produces a union type that is too complex to represent`。

### 递归深度控制

递归模板字面量类型（如嵌套的 `Split`、`Replace`）有大约 50 层的递归深度限制。日常使用不用担心，但如果处理超长字符串时需要注意。

## 高频面试题解析

### Q1：如何用模板字面量类型实现 `KebabCase` 类型？

**答案**：

```typescript
type KebabCase<S extends string> =
  S extends `${infer First}${infer Rest}`
    ? Rest extends Uncapitalize<Rest>
      ? `${Lowercase<First>}${KebabCase<Rest>}`
      : `${Lowercase<First>}-${KebabCase<Rest>}`
    : S;

type A = KebabCase<"HelloWorld">;      // "hello-world"
type B = KebabCase<"forEachItem">;     // "for-each-item"
type C = KebabCase<"HTMLElement">;     // "html-element"
```

### Q2：模板字面量类型的分布式特性是什么？有什么注意事项？

**答案**：
- 当模板中的占位符是联合类型时，会自动对每个联合成员分别展开后取并集
- 实际上是类型层面的"笛卡尔积"
- 注意事项：联合类型成员过多时可能超过编译器限制；`string | number | boolean` 中的 `string` 会展开为所有字符串，导致无意义的宽泛类型

### Q3：如何用模板字面量类型实现类型安全的 `trim`？

**答案**：

```typescript
type TrimLeft<S extends string> =
  S extends `${" " | "\n" | "\t"}${infer Rest}` ? TrimLeft<Rest> : S;

type TrimRight<S extends string> =
  S extends `${infer Rest}${" " | "\n" | "\t"}` ? TrimRight<Rest> : S;

type Trim<S extends string> = TrimRight<TrimLeft<S>>;

type Result = Trim<"  Hello World  ">;  // "Hello World"
```

## 总结与扩展

### 核心要点

1. **模板字面量类型** = 类型空间的字符串运算
2. **联合类型自动展开**（笛卡尔积）是最核心的特性
3. **`infer` + 模板字面量** = 类型层面的字符串模式匹配
4. 内置工具类型（Uppercase/Lowercase/Capitalize/Uncapitalize）提供基础字符串转换能力
5. 注意编译器性能边界（10万个联合成员上限）
