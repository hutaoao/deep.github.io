## 一句话概括

**面试高频题："手写 Partial、Pick、Exclude 这些内置工具类型"——本文把它们的"源码"展开讲通，让你从"会用"变成"能写、会解释"。**

## 背景

TypeScript 自带一套工具类型（Utility Types），定义在 `lib.es5.d.ts` 中，仅有寥寥数十行代码。绝大多数开发者能熟练使用 `Partial<T>`、`Pick<T, K>`，但面试时一被问到：

- "`Exclude<'a' | 'b', 'a'>` 为什么返回 `'b'`？"
- "`NonNullable<never>` 为什么是 `never`？"
- "手写 `MyReturnType`"

就答不上来了。原因不是不了解语法，而是**没理解分布式条件类型和 `infer` 这两个核心机制**。

本文的目标不是让你去读 TypeScript 编译器源码，而是**把每一行代码拆开、讲透，让你能在面试中清晰复述**。

## 前置概念（三句话讲清）

**1. 泛型约束**：`<T extends SomeType>` 要求 T 必须满足某种形状。

**2. 映射类型**：`{ [P in K]: T[P] }` 遍历联合类型 K 中的每个成员，逐个生成属性。

**3. 分布式条件类型**：当 `T extends U ? X : Y` 中的 `T` 是**裸类型参数**且为联合时，TypeScript 会**拆开联合，分别判断，再合并结果**。

```typescript
// 映射类型：遍历属性
type Clone<T> = { [P in keyof T]: T[P] };

// 分布式条件类型：拆开联合
type Distribute<T> = T extends string ? 'yes' : 'no';
type R = Distribute<'a' | 1>;  // 'yes' | 'no'  ✅ 分别判断
```

## 核心工具类型逐一手写

### 1. Partial / Required / Readonly（映射类型修饰符）

```typescript
// 所有属性变可选
type MyPartial<T> = {
    [P in keyof T]?: T[P];
};

// 所有属性变必选
type MyRequired<T> = {
    [P in keyof T]-?: T[P];
};

// 所有属性变只读
type MyReadonly<T> = {
    readonly [P in keyof T]: T[P];
};
```

**面试考点**：`-?` / `-readonly` 修饰符表示"去掉可选/去掉只读"。映射类型遍历 `keyof T` 时，TypeScript 会保留源类型的 `readonly` 和 `optional` 修饰符（同态映射），需要用 `-` 来移除。

### 2. Pick（选择部分属性）

```typescript
type MyPick<T, K extends keyof T> = {
    [P in K]: T[P];
};
```

**逐行解析**：
- `<K extends keyof T>`：约束 K 必须是 T 的属性名之子集
- `[P in K]`：遍历 K 中的每个键
- `T[P]`：取出对应的值类型

**面试手写**：这就是标准答案——三行搞定。

### 3. Omit（排除部分属性）

```typescript
// 利用 Pick + Exclude：先排除键名，再选取
type MyOmit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;
```

### 4. Record（构造键值对象）

```typescript
type MyRecord<K extends keyof any, T> = {
    [P in K]: T;
};
```

| 部分 | 含义 |
|------|------|
| `K extends keyof any` | K 必须是合法键名（`string \| number \| symbol` 的子集） |
| `[P in K]` | 遍历 K 中的每个成员，生成一个属性 |
| `: T` | 所有属性的值类型都是 T |

```typescript
// 使用示例
type CatInfo = MyRecord<'name' | 'age', string>;
// { name: string; age: string }
```

**面试对比**：`Record` 是固定键集合，`{ [key: string]: T }` 是索引签名（允许任意 string 键），后者类型更宽泛。

### 5. Exclude（从联合中排除类型）

```typescript
type MyExclude<T, U> = T extends U ? never : T;
```

**💡 这是面试最高频的手写题之一！**

**执行逻辑（以 `Exclude<'a' | 'b' | 'c', 'a'>` 为例）**：

分布式条件类型把 `'a' | 'b' | 'c'` 拆开分别判断：
1. `'a' extends 'a' ? never : 'a'` → `never`
2. `'b' extends 'a' ? never : 'b'` → `'b'`
3. `'c' extends 'a' ? never : 'c'` → `'c'`

合并结果：`never | 'b' | 'c'` → `'b' | 'c'`（`never` 在联合中被吸收）

**为什么这里要用裸类型参数？**

```typescript
// ✅ 分发：T 是裸类型参数
type ExcludeGood<T, U> = T extends U ? never : T;

// ❌ 不分发：T 被 [] 包裹
type ExcludeBad<T, U> = [T] extends [U] ? never : T;
type Test = ExcludeBad<'a' | 'b', 'a'>;  // 'a' | 'b'，没有排除！
```

### 6. Extract（从联合中提取类型）

```typescript
type MyExtract<T, U> = T extends U ? T : never;
```

Exclude 的镜像操作，仅交换 true/false 分支：

1. `'a' extends 'a' | 'f' ? 'a' : never` → `'a'`
2. `'b' extends 'a' | 'f' ? 'b' : never` → `never`
3. `'c' extends 'a' | 'f' ? 'c' : never` → `never`

合并：`'a' | never | never` → `'a'`

### 7. NonNullable（排除 null 和 undefined）

```typescript
type MyNonNullable<T> = T extends null | undefined ? never : T;
```

等价于 `Exclude<T, null | undefined>`。

### 8. ReturnType 和 Parameters（infer 提取）

```typescript
type MyReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R ? R : any;

type MyParameters<T extends (...args: any) => any> =
  T extends (...args: infer P) => any ? P : never;
```

**`infer` 的作用**：在条件类型的匹配过程中声明一个"类型变量"，TypeScript 会自动推断它的具体值。

```typescript
type Fn = (x: number, y: string) => boolean;

type Params = MyParameters<Fn>;  // [x: number, y: string]
type Ret = MyReturnType<Fn>;     // boolean
```

**面试追问**：`Parameters` 中泛型约束已经确保 `T` 是函数，所以条件类型的匹配必定成功，永远不会走到 `never` 分支。

## 实战案例

### 案例 1：用户状态分类

```typescript
type UserStatus = 'online' | 'offline' | 'away' | 'busy';
type ActiveStatus = 'online' | 'busy';

type ActiveUsers = Extract<UserStatus, ActiveStatus>;
// 'online' | 'busy'

type InactiveUsers = Exclude<UserStatus, ActiveStatus>;
// 'offline' | 'away'
```

### 案例 2：深度可选配置

```typescript
type DeepPartial<T> = {
    [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface Config {
    server: { host: string; port: number };
    debug: boolean;
}

type PartialConfig = DeepPartial<Config>;
// { server?: { host?: string; port?: number }; debug?: boolean }
```

### 案例 3：函数类型过滤器

```typescript
// 从联合函数类型中提取返回值为 string 的函数
type ReturnString<T> = T extends (...args: any[]) => infer R
    ? R extends string ? T : never
    : never;

type F1 = () => string;
type F2 = () => number;

type StringFuncs = ReturnString<F1 | F2>;  // () => string
```

## 面试必问：分布式条件类型 & never 传播（三句话讲清）

### 分布式条件类型的触发条件

当条件类型 `T extends U ? X : Y` 中左边的 `T` **是裸类型参数**时触发分发：

```typescript
// ✅ 分发
type Foo<T> = T extends string ? 'yes' : 'no';

// ❌ 不分发：被 [] 包裹
type Bar<T> = [T] extends [string] ? 'yes' : 'no';
```

### never 的"幽灵"行为

- **联合中**：`T | never` → `T`（被吸收）
- **条件类型中**：当 `T = never`，条件类型直接返回 `never`（因为空联合没有成员可供分发）

```typescript
type Check<T> = T extends string ? true : false;
type Result = Check<never>;  // never（不是 true 也不是 false！）
```

> **面试官最爱问**："`NonNullable<never>` 的结果是什么？" —— 答："never，因为 never 是空联合，分布式条件类型直接短路返回 never。"

## 高频面试题解析

### Q1：手写 MyPick

```typescript
type MyPick<T, K extends keyof T> = {
    [P in K]: T[P];
};
```

### Q2：手写 MyExclude 并解释为什么能排除

```typescript
type MyExclude<T, U> = T extends U ? never : T;
```

**分布式条件类型**把联合拆开分别判断，匹配的返回 `never` 被吸收，不匹配的保留。

### Q3：实现 MyReadonly2<T, K>（部分只读）

```typescript
type MyReadonly2<T, K extends keyof T = keyof T> = {
    readonly [P in K]: T[P];
} & {
    [P in Exclude<keyof T, K>]: T[P];
};
```

默认 `K = keyof T` 时等价于 `Readonly<T>`。

### Q4：为什么 Exclude<never, string> 是 never？

因为 `never` 是空联合类型，分布式条件类型对于空联合不进行任何分发，直接返回 `never`。

### Q5：手写一个深层排除 null/undefined 的工具类型

```typescript
type DeepNonNullable<T> = {
    [P in keyof T]-?: T[P] extends object
        ? DeepNonNullable<NonNullable<T[P]>>
        : NonNullable<T[P]>;
};
```

## 总结

| 工具类型 | 核心机制 | 面试手写关键 |
|---------|---------|------------|
| `Partial/Required/Readonly` | 映射类型 + 修饰符 | `-?` / `-readonly` |
| `Record<K,T>` | 映射类型 | `K extends keyof any` |
| `Pick<T,K>` | 映射类型 + 约束 | `K extends keyof T` |
| `Omit<T,K>` | Exclude + Pick 组合 | `Pick<T, Exclude<keyof T, K>>` |
| `Exclude<T,U>` | 分布式条件类型 | 裸类型参数才会分发 |
| `Extract<T,U>` | 分布式条件类型 | 镜像 Exclude |
| `NonNullable<T>` | 分布式条件类型 | 排除 null/undefined |
| `ReturnType<T>` | `infer` 推断 | `infer R` 在返回值位置 |
| `Parameters<T>` | `infer` 推断 | `infer P` 在参数位置 |
