## 一句话概括

**协变与逆变是类型系统中的父子关系传递规则**：子类型可以赋值给父类型的叫协变，父类型可以赋值给子类型的叫逆变——这是理解 TypeScript 类型安全的核心。

## 背景

协变与逆变是类型系统中最容易被忽视但极其重要的概念。

面试中常见问题：
- "什么是协变？什么是逆变？"
- "为什么函数参数是逆变的而返回值是协变的？"
- "TypeScript 中如何保证类型安全？"

理解这两个概念，才能真正掌握 TypeScript 的类型系统，写出类型安全的代码。

## 概念与定义

### 什么是协变（Covariance）？

**协变**：子类型可以赋值给父类型。

```typescript
// 动物是猫的父类型
class Animal {
  name: string = '动物';
}

class Cat extends Animal {
  age: number = 1;
}

// 协变：Cat 可以赋值给 Animal
const animal: Animal = new Cat();  // ✅ 正确
```

### 什么是逆变（Contravariance）？

**逆变**：父类型可以赋值给子类型（仅限函数参数）。

```typescript
// 动物是猫的父类型
class Animal {
  name: string = '动物';
}

class Cat extends Animal {
  age: number = 1;
}

// 逆变：Animal 可以赋值给 Cat 的函数参数
type CatFunc = (cat: Cat) => void;
type AnimalFunc = (animal: Animal) => void;

const catFunc: CatFunc = (cat) => console.log(cat.age);
const animalFunc: AnimalFunc = catFunc;  // ✅ 逆变允许
```

### 函数参数的逆变与返回值的协变

```typescript
// 父类型
type Parent = {
  name: string;
  age: number;
};

// 子类型
type Child = {
  name: string;
  age: number;
  hobby: string;
};

// 函数参数：逆变（父类型可以赋值给子类型的参数）
type ParentToChild = (parent: Parent) => Child;
type ChildToParent = (child: Child) => Parent;

// 正确：父函数可以赋值给子函数（参数逆变）
const parentFn: ChildToParent = (child) => ({ name: child.name, age: child.age });
const childFn: ParentToChild = parentFn;  // ✅

// 错误：子函数不能赋值给父函数
// const wrongFn: ChildToParent = (child) => ({ name: child.name, age: child.age, hobby: 'play' });
// const fn: ParentToChild = wrongFn;  // ❌
```

## 最小示例

### 数组的协变

```typescript
// TypeScript 中数组是协变的
const animals: Animal[] = [];  // Animal[]
const cats: Cat[] = [new Cat()];  // Cat[]

animals.push(new Cat());  // ✅ Cat 是 Animal 的子类型，可以 push 进 Animal[]
```

### 函数的协变与逆变

```typescript
// 函数返回值的协变
type GetAnimal = () => Animal;
type GetCat = () => Cat;

const getAnimal: GetAnimal = () => new Animal();
const getCat: GetCat = getAnimal;  // ✅ 返回值协变

// 函数参数的逆变
type TakeAnimal = (animal: Animal) => void;
type TakeCat = (cat: Cat) => void;

const takeCat: TakeCat = (cat) => console.log(cat);
const takeAnimal: TakeAnimal = takeCat;  // ✅ 参数逆变
```

## 核心知识点拆解

### 1. 类型安全的核心规则

- **返回值协变**：子类型的返回值可以赋值给父类型的返回值
- **参数逆变**：父类型的参数可以��值给子类型的参数
- **原理**：函数调用时，调用者提供参数（需要父类型），返回时接收返回值（需要子类型）

### 2. TypeScript 的处理方式

```typescript
// TypeScript 默认 strictFunctionTypes 开启
class Animal {
  name: string;
}

class Cat extends Animal {
  age: number;
}

// 函数参数逆变检查
type Fn1 = (cat: Cat) => Cat;
type Fn2 = (animal: Animal) => Animal;

// 正确
const fn2: Fn2 = (a) => a;
const fn1: Fn1 = fn2;  // ✅ 参数逆变

// 错误（strictFunctionTypes 下）
// const fn2Wrong: Fn2 = (a) => a;
// const fn1Wrong: Fn1 = fn2Wrong;  // ❌ 参数协变不允许
```

### 3. 泛型中的协变与逆变

```typescript
// 泛型默认是双向协变的（除非 strictFunctionTypes）
type Container<T> = {
  value: T;
};

// 协变
type AnimalContainer = Container<Animal>;
type CatContainer = Container<Cat>;

const catContainer: CatContainer = { value: new Cat() };
const animalContainer: AnimalContainer = catContainer;  // ✅ 协变
```

## 实战案例

### 实际开发中的应用

```typescript
// 事件处理系统的类型设计
type EventHandler<T> = (data: T) => void;

// 用户事件
interface UserEvent {
  userId: string;
  action: string;
}

// 扩展事件
interface UserEventWithTime extends UserEvent {
  timestamp: number;
}

// 正确的处理器类型
type Handler = EventHandler<UserEvent>;
type ExtendedHandler = EventHandler<UserEventWithTime>;

// 扩展处理器可以赋值给基础处理器（协变）
const extendedHandler: ExtendedHandler = (event) => {
  console.log(event.timestamp);
};

const baseHandler: Handler = extendedHandler;  // ✅

// 实际使用
function emitEvent(handler: Handler, data: UserEvent) {
  handler(data);
}

emitEvent(baseHandler, { userId: '123', action: 'click' });
```

## 底层原理

### TypeScript 编译器的处理

1. **类型检查阶段**：在编译时进行类型检查
2. **函数类型比较**：比较函数参数和返回值的类型关系
3. **双向协变开关**：`strictFunctionTypes` 控制是否允许函数参数协变

```typescript
// 源码位置：src/compiler/checker.ts
// 函数类型兼容性检查核心逻辑：

function isCallableType(typeA, typeB) {
  // 返回值：协变检查
  const returnCompatible = isAssignable(typeB.return, typeA.return);
  
  // 参数：逆变检查
  const paramCompatible = isAssignable(typeA.params, typeB.params);
  
  return returnCompatible && paramCompatible;
}
```

### 逆变的数学解释

```
如果 A ⊆ B（ A 是 B 的子类型）：

协变：F(A) ⊆ F(B)  // 子类型的函数返回值是子类型
逆变：F(B) ⊆ F(A)  // 父类型的函数参数是子类型

这与函数类型的契约有关：
- 调用者提供参数（需要"更大"的类型）
- 函数返回结果（需要"更小"的类型）
```

## 高频面试题解析

### Q1：为什么函数参数是逆变的？

**答案**：考虑类型安全。函数被调用时，调用者传入的参数类型决定了函数能做什么。如果函数期望 `Cat`，传入 `Animal`（更宽泛），函数可能会用到 `Cat` 特有的属性导致错误。所以需要传入更具体的类型（子类型），这就是逆变。

### Q2：TypeScript 如何开启逆变检查？

**答案**：在 `tsconfig.json` 中开启 `strictFunctionTypes: true`（或 `strict: true`）。

```json
{
  "compilerOptions": {
    "strict": true,
    "strictFunctionTypes": true
  }
}
```

### Q3：协变和逆变的实际应用场景？

**答案**：
- **事件系统**：扩展事件处理器可以赋值给基础事件处理器
- **泛型容器**：子类型容器可以赋值给父类型容器
- **响应式库**：RxJS 等库的类型设计
