> 一句话概括：Vue 3 的响应式系统基于 ES6 原生 `Proxy` 代理机制，在对象访问（get）和修改（set）时自动追踪依赖并触发更新，相比 Vue 2 的 `Object.defineProperty` 实现了更全面、更高效的响应式能力。

## 背景

Vue 2 的响应式系统基于 `Object.defineProperty`，这个 API 在 Vue 2 时代是唯一的选择，但它有几个根本性限制：无法检测数组下标变化、无法检测对象属性的添加和删除（除非使用 `Vue.set`）、需要递归遍历所有属性。Vue 3 选用了 ES6 的 `Proxy` 作为响应式系统的底层实现，这些问题从根本上得到了解决。

## 概念与定义

### Proxy 是什么

`Proxy` 是 ES6 引入的元编程（Metaprogramming）特性，它允许你创建一个对象的**代理**，拦截并自定义该对象的底层操作（属性访问、赋值、函数调用等）：

```typescript
const target = { name: 'Alice', age: 18 };

const proxy = new Proxy(target, {
  // 拦截属性读取
  get(target, key, receiver) {
    console.log(`读取 ${String(key)}`);
    return Reflect.get(target, key, receiver);
  },
  // 拦截属性赋值
  set(target, key, value, receiver) {
    console.log(`设置 ${String(key)} = ${value}`);
    return Reflect.set(target, key, value, receiver);
  }
});

proxy.name;       // 打印：读取 name
proxy.age = 30;   // 打印：设置 age = 30
```

### Proxy vs Object.defineProperty

| 特性 | Object.defineProperty | Proxy |
|------|---------------------|-------|
| 拦截方式 | 属性级别（需要遍历每个属性） | 对象级别（一个代理拦截所有操作） |
| 新增属性 | 手动 defineProperty | 自动拦截 |
| 删除属性 | 手动处理 | `deleteProperty` 拦截 |
| 数组下标 | 需要特殊处理 | 自动拦截 |
| 深层嵌套 | 需要递归包装 | Proxy 递归自动生效 |
| 性能 | 每个属性单独处理 | 一次性代理整棵树 |

## 核心知识点拆解

### 1. Vue 3 的 reactive 函数

```typescript
import { reactive } from 'vue';

// reactive 将普通对象转为响应式代理
const state = reactive({
  count: 0,
  user: {
    name: 'Alice',
    profile: { age: 18 }
  }
});

// 自动追踪：模板中使用 state.count 会建立依赖
state.count++;  // 触发更新

// 深层响应：嵌套对象也会自动代理
state.user.profile.age = 30;  // 也会触发更新
```

### 2. get 拦截器（依赖收集）

当访问响应式对象的属性时，`get` 拦截器被触发。此时记录当前正在执行的副作用（effect），将当前属性作为依赖关联起来：

```typescript
// 简化的依赖收集逻辑
let activeEffect: Function | null = null;

function track(target, key) {
  if (activeEffect) {
    // 将 activeEffect 关联到 target.key
    depsMap.get(target)?.get(key)?.add(activeEffect);
  }
}

const reactiveHandler = {
  get(target, key, receiver) {
    const result = Reflect.get(target, key, receiver);
    track(target, key);  // 收集依赖
    // 如果结果是对象，递归代理（懒代理）
    if (isObject(result)) {
      return reactive(result);
    }
    return result;
  }
};
```

### 3. set 拦截器（触发更新）

当修改响应式对象的属性时，`set` 拦截器被触发，找出所有依赖当前属性的副作用并执行：

```typescript
function trigger(target, key) {
  const effects = depsMap.get(target)?.get(key);
  if (effects) {
    effects.forEach(effect => effect());
  }
}

const reactiveHandler = {
  set(target, key, value, receiver) {
    const oldValue = target[key];
    const result = Reflect.set(target, key, value, receiver);
    if (oldValue !== value) {
      trigger(target, key);  // 触发更新
    }
    return result;
  }
};
```

### 4. Reflect 对象的作用

`Reflect` 是 ES6 引入的操作对象的标准化 API。使用 `Reflect` 而非直接操作 `target`，有以下几个重要原因：

```typescript
// ❌ 错误做法：直接操作 target
const result = target[key];  // 不保持 this 绑定

// ✅ 正确做法：使用 Reflect
const result = Reflect.get(target, key, receiver);
// Reflect 保持原型链正确：
// 当 target 有 getter，且 getter 依赖 this 时，
// receiver 正确传递 this 引用
```

### 5. 深层嵌套的自动响应

Vue 3 的 Proxy 是**懒代理**的：只有被访问到的嵌套对象才会被包装成响应式：

```typescript
const state = reactive({ a: { b: { c: 1 } } });

// 访问时自动代理
state.a.b.c = 2;  // 触发更新

// 未访问的深层路径不会被代理（节省性能）
```

## 实战案例

### 案例：手写简易 reactive 函数

```typescript
// 依赖映射表：target -> (key -> [effect])
const depsMap = new WeakMap<object, Map<string | symbol, Set<Function>>>();

// 获取依赖集合
function getDep(target, key) {
  let dep = depsMap.get(target);
  if (!dep) {
    dep = new Map();
    depsMap.set(target, dep);
  }
  let effects = dep.get(key);
  if (!effects) {
    effects = new Set();
    dep.set(key, effects);
  }
  return effects;
}

// 依赖收集
function track(target, key) {
  if (activeEffect) {
    getDep(target, key).add(activeEffect);
  }
}

// 触发更新
function trigger(target, key) {
  getDep(target, key).forEach(effect => effect());
}

// 创建响应式对象
function reactive(target) {
  return new Proxy(target, {
    get(target, key, receiver) {
      const result = Reflect.get(target, key, receiver);
      track(target, key);
      // 深层代理：嵌套对象也变成响应式
      if (result !== null && typeof result === 'object') {
        return reactive(result);
      }
      return result;
    },
    set(target, key, value, receiver) {
      const oldValue = target[key];
      const result = Reflect.set(target, key, value, receiver);
      if (oldValue !== value) {
        trigger(target, key);
      }
      return result;
    },
    deleteProperty(target, key) {
      const hadKey = key in target;
      const result = Reflect.deleteProperty(target, key);
      if (hadKey && result) {
        trigger(target, key);
      }
      return result;
    }
  });
}

// 副作用函数
let activeEffect = null;
function effect(fn: () => void) {
  activeEffect = fn;
  fn();
  activeEffect = null;
}

// 测试
const state = reactive({ count: 0, name: 'Alice' });

effect(() => {
  console.log(`count changed: ${state.count}`);
  console.log(`name: ${state.name}`);
});

state.count++;  // 触发副作用，打印 "count changed: 1"
// state.name = 'Bob';  // 触发副作用
```

### 案例：computed 的实现

```typescript
function computed(getter) {
  let value;
  let dirty = true;  // 脏标记：是否需要重新计算

  const effect = () => { value = getter(); };

  const runner = () => {
    if (dirty) {
      dirty = false;
      trigger(value, 'value');  // 通知依赖方
    }
  };

  // 将 computed 包装为一个响应式对象
  return {
    get value() {
      track(value, 'value');  // 建立依赖
      if (dirty) {
        dirty = false;
        value = getter();
      }
      return value;
    }
  };
}
```

## 底层原理

### Vue 3 响应式的完整数据结构

Vue 3 内部使用以下数据结构管理响应式依赖：

```typescript
// 全局 WeakMap: target -> Map<key, Set<ReactiveEffect>>
type Dep = Set<ReactiveEffect>;
type KeyToDepMap = Map<string | symbol, Dep>;
type TargetMap = WeakMap<object, KeyToDepMap>;

const targetMap: TargetMap = new WeakMap();
```

**依赖收集时机**：在 `get` 拦截器中，通过 `track` 函数将当前活跃的 `ReactiveEffect` 注册到 `targetMap` 中。

**触发更新时机**：在 `set` 拦截器中，通过 `trigger` 函数找到 `targetMap[target][key]` 中所有注册的 `ReactiveEffect` 并执行。

### Proxy 的 Reflect 处理

使用 `Reflect` 有三个关键原因：

1. **保持正确的 this 绑定**：当对象有 getter/setter，且内部引用 `this` 时，`receiver` 确保 `this` 指向代理对象而非原始对象
2. **行为一致性**：`Reflect` 的返回值与 `Proxy` 拦截器期望的返回值语义完全一致
3. **完整性**：`Reflect` 提供了所有 `Object` 操作的一一对应方法

### Vue 3 的 Proxy 陷阱

```typescript
// 边界情况 1：直接替换整个对象
const state = reactive({ count: 0 });
state = reactive({ count: 10 });  // ❌ 不能重新赋值，reactive 不会自动套新代理

// 正确做法：
const state = reactive({ count: 0 });
Object.assign(state, { count: 10 });  // ✅ 修改现有属性

// 边界情况 2：原始值（不是对象）
const count = ref(0);  // ref 处理原始值，Proxy 不处理原始值
```

## 高频面试题解析

### Q1：Vue 3 为什么用 Proxy 而不是 Object.defineProperty？

**答：** 根本原因有三个：① **Proxy 是对象级别的拦截**，不需要对每个属性单独处理，新增属性自动被拦截，而 `defineProperty` 需要在已知属性上显式调用；② **Proxy 支持数组下标和类数组成员自动拦截**，而 `defineProperty` 对数组的拦截有性能问题；③ **Proxy 支持 `deleteProperty`、`has`、`ownKeys` 等拦截器**，完整覆盖了对象的所有操作。这些能力组合在一起，使得 Vue 3 可以用更少的代码实现更完整的响应式系统。

### Q2：Proxy 的懒代理（lazy proxy）是如何工作的？为什么这样设计？

**答：** Vue 3 在 `get` 拦截器中判断返回值是否为对象，如果是则递归调用 `reactive()` 将其包装为响应式代理。这就是懒代理——只有真正被访问到的嵌套对象才会被代理。好处：性能优化，不必在初始化时递归遍历整棵树；避免循环引用导致的无限递归。

### Q3：reactive 和 ref 的区别是什么？

**答：** `reactive` 将整个对象代理为响应式，内部通过 `Proxy` 实现；`ref` 包装原始值为响应式，内部通过一个带有 `get value()` 和 `set value()` 的对象实现。`ref` 的优势是可以处理原始类型（`number`、`string` 等），而 `reactive` 只能处理对象。访问 `ref` 的值需要 `.value`，但模板中会自动解包。

### Q4：Vue 3 中 ref 对对象类型的处理有什么特殊之处？

**答：** 当 `ref` 的值是对象时，Vue 3 会自动调用 `reactive()` 将其内部值包装为响应式：

```typescript
const count = ref({ num: 0 });
count.value.num++;  // ✅ 自动触发更新（内部调用 reactive）

// 但 ref 本身仍需要 .value 访问
// 模板中：{{ count.num }}（自动解包）
```

## 总结与扩展

### 核心要点

1. Vue 3 响应式基于 ES6 `Proxy`，拦截 `get`/`set`/`deleteProperty` 等操作
2. `get` 中做依赖收集（track），`set` 中触发更新（trigger）
3. `Reflect` 用于保持正确的 `this` 绑定和原型链行为
4. 懒代理设计：只有访问到的嵌套对象才会被递归包装
5. `reactive` 用于对象，`ref` 用于原始值（内部对象也用 reactive）
6. `Proxy` 解决了 Vue 2 `defineProperty` 的三大限制：数组下标、新增属性、删除属性

### 实际工程建议

- 避免响应式对象的重新赋值，使用 `Object.assign` 或解构赋值
- 避免使用 `JSON.parse(JSON.stringify())` 拷贝响应式对象（会丢失代理），改用 `toRaw` 获取原始对象后再拷贝
- 在大型列表中使用 `shallowRef` 避免深层代理的性能开销
- 深度嵌套的响应式对象在大量数据场景下可能带来 GC 压力，使用 `markRaw` 标记不需要响应化的数据
