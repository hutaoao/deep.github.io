> 一句话概括：TypeScript 枚举是唯一一种既有"类型"约束，又会在编译后生成真实运行时代码的类型语法，其背后是编译器精心设计的状态机模式。

## 背景

在强类型语言中，枚举（Enum）是表达"一组具名常量"的经典手段。TypeScript 在 JavaScript 的基础上引入了枚举语法糖，但很多人只把它当成普通的类型标注，忽略了它对运行时的影响。理解枚举的编译结果，是写出高效 TypeScript 代码、避免踩坑的重要前提。

## 概念与定义

### 什么是枚举

枚举是 TypeScript 中的一种语法，用于定义一个包含一组具名常量值的类型。例如：

```typescript
enum Direction {
  Up,    // 默认值为 0
  Down,  // 自动递增为 1
  Left,  // 自动递增为 2
  Right  // 自动递增为 3
}
```

### 枚举的两大类型

| 类型 | 关键字 | 特点 | 编译结果 |
|------|--------|------|----------|
| 数字枚举 | `enum` | 可双向映射，生成运行时代码 | 双向 Map 对象 |
| 字符串枚举 | `enum` + 字符串值 | 单向映射，生成更少代码 | 仅正映射对象 |
| 常量枚举 | `const enum` | 编译时内联，删除枚举定义 | 无运行时代码 |

## 最小示例

### 数字枚举

```typescript
enum Status {
  Pending = 0,
  Success = 1,
  Error = 2
}

// 正向映射
Status.Pending  // 0
Status[0]       // "Pending" —— 反向映射也可用

// 使用
const res = { code: 0, msg: 'ok' }
if (res.code === Status.Success) {
  // ...
}
```

### 字符串枚举

```typescript
enum Direction {
  Up = 'UP',
  Down = 'DOWN'
}

// 正向映射
Direction.Up  // 'UP'
// Direction['UP']  ❌ 反向映射不存在
```

### const 枚举

```typescript
const enum Color {
  Red = 'red',
  Blue = 'blue'
}

// 编译后：直接内联字面量
const color = Color.Red  // → const color = 'red'
// 枚举对象本身被完全删除
```

## 核心知识点拆解

### 1. 枚举值与自动递增

```typescript
enum E {
  A,    // 0
  B,    // 1
  C = 5,
  D,    // 6（基于前一个值自动递增）
  E     // 7
}
```

枚举值支持自定义初始值，后续未赋值的成员自动在前一个数值的基础上 +1。

### 2. 枚举的编译结果（重点）

**数字枚举编译为：**

```javascript
"use strict";
var Status;
(function (Status) {
    Status[Status["Pending"] = 0] = "Pending";
    Status[Status["Success"] = 1] = "Success";
    Status[Status["Error"] = 2] = "Error";
})(Status || (Status = {}));
```

拆解这个 IIFE 的作用：

```javascript
// 展开后的赋值等价于：
Status["Pending"] = 0;
Status[0] = "Pending";   // 反向映射：数值 → 字符串名

Status["Success"] = 1;
Status[1] = "Success";

Status["Error"] = 2;
Status[2] = "Error";
```

这就是为什么 `Status[0]` 能得到字符串 "Pending"——编译器同时生成了正向和反向两个方向的映射。

**字符串枚举编译为：**

```javascript
"use strict";
var Direction;
(function (Direction) {
    Direction["Up"] = "UP";
    Direction["Down"] = "DOWN";
})(Direction || (Direction = {}));
```

只有正向映射，没有反向映射。

**常量枚举编译为：**

```javascript
// 编译后：完全消失
const color = 'red'  // Color.Red 被直接内联
```

### 3. const enum 的优势与限制

```typescript
const enum Color {
  Red, Green, Blue
}

// ✅ 优势：编译后完全无运行时代码
// ❌ 限制：不能用于需要运行时反射的场景
// console.log(Color)  // 编译错误
```

### 4. 枚举与联合类型

当枚举所有成员都是字面量值时，枚举本身会成为**字面量类型**：

```typescript
enum E { A, B }
type T = E  // 枚举类型
type U = keyof typeof E  // 提取成员名

function f(x: E): void {}

// f(E.A)  ✅
// f(0)    ✅（数字枚举可赋值数值）
// f(99)   ❌
```

但如果你给枚举成员赋值为字面量，枚举类型就变成了具体的字面量联合：

```typescript
enum Direction {
  Up = 'UP',
  Down = 'DOWN'
}

type Dir = Direction  // = 'UP' | 'DOWN'
```

## 实战案例

### 案例：状态机管理

```typescript
enum OrderStatus {
  Created = 'CREATED',
  Paid = 'PAID',
  Shipped = 'SHIPPED',
  Completed = 'COMPLETED',
  Cancelled = 'CANCELLED'
}

type StatusConfig = {
  label: string;
  color: string;
  nextAllowed: OrderStatus[];
};

const statusConfig: Record<OrderStatus, StatusConfig> = {
  [OrderStatus.Created]: {
    label: '已创建',
    color: '#999',
    nextAllowed: [OrderStatus.Paid, OrderStatus.Cancelled]
  },
  [OrderStatus.Paid]: {
    label: '已支付',
    color: '#1890ff',
    nextAllowed: [OrderStatus.Shipped, OrderStatus.Cancelled]
  },
  // ...
};

function transition(current: OrderStatus, next: OrderStatus): boolean {
  const config = statusConfig[current];
  return config.nextAllowed.includes(next);
}
```

### 案例：使用 const enum 优化 Bundle

```typescript
// 业务代码
const enum HttpStatus {
  OK = 200,
  NotFound = 404,
  ServerError = 500
}

// 普通 enum: 生成 Status 对象，增加 bundle 体积
// const enum: Status 对象消失，只留下字面量
```

在大型前端项目中，大量使用普通枚举会显著增加运行时代码体积。将不会在运行时被引用的枚举改为 `const enum` 是优化的有效手段。

## 底层原理

### 编译器枚举处理流程

1. **解析阶段**：TypeScript 解析器将枚举声明解析为 `EnumDeclaration` AST 节点
2. **类型检查**：验证枚举成员赋值的类型一致性
3. **代码生成**：
   - 普通枚举：生成 IIFE 包裹的映射对象，包含正反双向映射
   - const 枚举：编译器在每个引用点内联枚举值，删除枚举定义
4. **esm/comonjs 模块化**：枚举对象的模块导出遵循项目设置的 module 规则

### 枚举的内存模型

```javascript
// 编译后枚举对象的内存结构
Status = {
  '0': 'Pending',    // 反向映射（数值键）
  '1': 'Success',
  '2': 'Error',
  Pending: 0,         // 正向映射（字符串键）
  Success: 1,
  Error: 2
}
```

**关键点**：数字枚举同时维护了 `string→number` 和 `number→string` 两条映射链，这就是为什么枚举既能用于对象键访问，也能用于数组索引。

## 高频面试题解析

### Q1：const enum 和普通 enum 的区别是什么？

**答：** 普通 enum 在编译后会生成运行时的 JavaScript 对象（IIFE 模式），同时包含正向和反向（数字枚举）两条映射；const enum 在编译时完全被消除，每个引用点直接替换为对应的字面量值，不生成任何运行时代码。因此 const enum 有更好的编译后性能，但无法用于需要运行时反射的场景。

### Q2：为什么有时候 `Enum['Key']` 能访问，有时候不行？

**答：** 取决于枚举成员的类型。如果是**数字枚举**，无论用字符串键还是数字键都可以访问（双向映射）。如果是**字符串枚举**，只能通过字符串键访问，数字索引会返回 `undefined`。常量枚举则完全不存在于运行时，根本无法作为对象访问。

### Q3：可以用数字给字符串枚举赋值吗？

**答：** 不可以。TypeScript 会在编译时报错：`Enum member must have string value`。字符串枚举的每个成员必须被赋值为字符串字面量或常量表达式。

### Q4：如何获取枚举的所有成员名？

**答：** 利用 `Object.keys()` 可以枚举对象的所有键：

```typescript
enum Status { Pending, Success, Error }

// 获取所有成员名（字符串键）
const names = Object.keys(Status).filter(k => isNaN(Number(k)))
// ['Pending', 'Success', 'Error']

// 获取所有数值
const values = Object.values(Status).filter(v => !isNaN(Number(v)))
// [0, 1, 2]
```

## 总结与扩展

### 核心要点总结

1. **数字枚举**会生成双向映射（正映射 + 反映射），编译为 IIFE 对象
2. **字符串枚举**只生成正向映射，代码量更少
3. **const enum**编译时内联，完全消除运行时代码，是性能敏感场景的首选
4. 枚举成员赋值后，未赋值的成员自动递增
5. 字面量枚举值使枚举类型退化为字面量联合类型

### 实际工程建议

- 对**只在编译时使用**的枚举（如配置 key），使用 `const enum`
- 对**需要在运行时访问枚举名/值**的场景，使用普通数字枚举
- 对**状态语义明确**的场景，优先使用**字符串枚举**（更安全，无反向映射歧义）
- 避免在枚举中使用非 0/1 起始值，容易引发团队理解歧义
