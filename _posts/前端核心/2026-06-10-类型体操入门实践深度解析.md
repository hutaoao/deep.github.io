## 一句话概括

**类型体操是在 TypeScript 类型系统中进行编程的实践**：利用条件类型、映射类型、模板字面量类型和递归，在类型层面完成复杂的数据转换——这不仅是面试考点，更是编写类型安全库的核心能力。

## 背景

"类型体操"（Type Gymnastics）是 TypeScript 社区对高级类型编程的形象称呼。随着 TypeScript 类型系统的不断完善（条件类型、infer、模板字面量类型、递归类型），我们可以在类型层面完成越来越复杂的计算。

面试中常见问题：
- "如何判断一个类型是否为联合类型？"
- "如何将联合类型转为交叉类型？"
- "如何深度提取对象的属性路径？"
- "如何实现 DeepReadonly？"

掌握类型体操的核心理念，才能真正理解 TypeScript 的威力。

## 概念与定义

### 什么是类型体操？

类型体操本质上是在 TypeScript 的**类型空间**（而非值空间）中进行逻辑运算。它使用以下核心工具：

| 工具 | 说明 | 示例 |
|------|------|------|
| 条件类型 | `extends ? :` 做类型判断 | `T extends string ? "yes" : "no"` |
| `infer` | 在条件类型中提取类型 | `T extends Array<infer U> ? U : never` |
| 映射类型 | 遍历对象属性并转换 | `{[K in keyof T]: T[K]}` |
| 模板字面量类型 | 字符串类型运算 | `` `${A}-${B}` `` |
| 递归类型 | 类型引用自身 | `type Deep<T> = T extends object ? {[K in keyof T]: Deep<T[K]>} : T` |

### 类型体操的三个层次

```
入门层：理解条件类型、infer、映射类型
        ↓
进阶层：递归类型、联合类型操作、模板字面量模式匹配
        ↓
高级层：类型层面的数据结构模拟（元组操作、数值运算、类型级编程）
```

## 最小示例

### 示例 1：递归类型的基本用法

```typescript
// DeepReadonly - 递归地将所有属性设为只读
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? DeepReadonly<T[K]>
    : T[K];
};

interface User {
  name: string;
  address: {
    city: string;
    zip: number;
  };
}

type ReadonlyUser = DeepReadonly<User>;
// {
//   readonly name: string;
//   readonly address: {
//     readonly city: string;
//     readonly zip: number;
//   };
// }
```

### 示例 2：联合类型转交叉类型

```typescript
// 核心：利用函数参数逆变的特性
type UnionToIntersection<U> =
  (U extends any ? (k: U) => void : never) extends (k: infer I) => void
    ? I
    : never;

type Union = { a: string } | { b: number };
type Intersection = UnionToIntersection<Union>;
// { a: string; b: number }
```

### 示例 3：对象路径提取

```typescript
type ObjectPaths<T> = T extends object
  ? {
      [K in keyof T & string]: T[K] extends object
        ? K | `${K}.${ObjectPaths<T[K]>}`
        : K;
    }[keyof T & string]
  : never;

type Paths = ObjectPaths<{ a: { b: { c: number } } }>;
// "a" | "a.b" | "a.b.c"
```

## 核心知识点拆解

### 1. 递归类型（Recursive Conditional Types）

TypeScript 4.1+ 支持在条件类型中递归引用自身（有限制）：

```typescript
// 将嵌套的对象类型扁平化
type Flatten<T> = T extends Array<infer U>
  ? Flatten<U>
  : T;

type A = Flatten<string[][][]>;  // string

// 深度 Partial
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object
    ? DeepPartial<T[K]>
    : T[K];
};

// 深度 Required
type DeepRequired<T> = {
  [K in keyof T]-?: T[K] extends object
    ? DeepRequired<T[K]>
    : T[K];
};
```

**注意**：TypeScript 对递归深度有限制（约 50 层），并会检查是否有明确的终止条件。

### 2. 联合类型转交叉类型（Union to Intersection）

这是类型体操中最经典也最令人困惑的技巧之一：

```typescript
type UnionToIntersection<U> =
  (U extends any ? (k: U) => void : never) extends (k: infer I) => void
    ? I
    : never;
```

**原理拆解**：

```typescript
// 步骤 1：将联合类型的每个成员放入函数参数位置（逆变位置）
// U = { a: string } | { b: number }
// 展开为：((k: { a: string }) => void) | ((k: { b: number }) => void)

// 步骤 2：利用函数参数逆变，推断出交叉类型
// ((k: {a:string}) => void) | ((k: {b:number}) => void) extends (k: infer I) => void
// 根据逆变规则，I 必须是 { a: string } & { b: number }
```

**核心原理**：函数参数是逆变的，所以当多个函数类型的联合要赋值给一个函数类型时，参数位置推断出的类型会自动取交集。

### 3. 判断是否为联合类型

```typescript
// 利用分布式条件类型的特性
type IsUnion<T, U = T> =
  [T] extends [never]
    ? false
    : T extends U
      ? [U] extends [T]
        ? false
        : true
      : false;

type Test1 = IsUnion<string>;           // false
type Test2 = IsUnion<string | number>;  // true
type Test3 = IsUnion<never>;            // false
```

**关键技巧**：`[T]` 和 `[U]` 的方括号包装阻止了分布式行为，使我们能够比较完整的联合类型。

### 4. 联合类型的遍历

利用**分布式条件类型**逐一遍历联合类型的成员：

```typescript
// 提取联合类型中的最后一个成员
type LastOfUnion<T> =
  UnionToIntersection<
    T extends any ? () => T : never
  > extends () => infer R
    ? R
    : never;

// 联合类型转元组
type UnionToTuple<T, Result extends any[] = []> =
  [T] extends [never]
    ? Result
    : UnionToTuple<Exclude<T, LastOfUnion<T>>, [LastOfUnion<T>, ...Result]>;
```

### 5. 对象路径提取

```typescript
// 提取嵌套对象的完整属性路径
type PathImpl<T, K extends keyof T> =
  K extends string
    ? T[K] extends Record<string, any>
      ? K | `${K}.${PathImpl<T[K], keyof T[K]>}`
      : K
    : never;

type Path<T> = PathImpl<T, keyof T>;
```

## 实战案例

### 案例 1：类型安全的前端路由参数提取

```typescript
// 定义路由结构
interface Routes {
  "/user/:id": { id: string };
  "/user/:id/posts": { id: string };
  "/post/:postId": { postId: string };
  "/post/:postId/comments": { postId: string; commentId?: never };
  "/post/:postId/comments/:commentId": { postId: string; commentId: string };
}

// 提取路由参数类型
type ExtractRouteParams<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? { [K in Param]: string } & ExtractRouteParams<`/${Rest}`>
    : T extends `${string}:${infer Param}`
      ? { [K in Param]: string }
      : {};

// 使用
type Params1 = ExtractRouteParams<"/user/:id">;
// { id: string }

type Params2 = ExtractRouteParams<"/post/:postId/comments/:commentId">;
// { postId: string; commentId: string }
```

### 案例 2：深度可选的表单校验类型

```typescript
// 深度 Partial + 深度 Required 组合
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object ? DeepPartial<T[K]> : T[K];
};

type DeepRequired<T> = {
  [K in keyof T]-?: T[K] extends object ? DeepRequired<T[K]> : T[K];
};

// 表单数据：创建时部分字段可选，编辑时全部必填
interface FormData {
  user: {
    name: string;
    profile: {
      avatar: string;
      bio: string;
    };
  };
  settings: {
    theme: "light" | "dark";
    language: string;
  };
}

type CreateFormData = DeepPartial<FormData>;
type EditFormData = DeepRequired<FormData>;
```

### 案例 3：API 响应类型的递归转换

```typescript
// 将 API 返回的日期字符串转为 Date 类型
type DeepConvertDates<T> = {
  [K in keyof T]: T[K] extends string
    ? T[K] extends `${number}-${number}-${number}`
      ? Date
      : T[K]
    : T[K] extends object
      ? DeepConvertDates<T[K]>
      : T[K];
};

interface UserResponse {
  name: string;
  birthDate: "1990-01-01";
  createdAt: "2023-06-15";
  profile: {
    lastLogin: "2024-01-20";
  };
}

type User = DeepConvertDates<UserResponse>;
// { name: string; birthDate: Date; createdAt: Date; profile: { lastLogin: Date } }
```

## 底层原理

### 分布式条件类型的工作原理

```typescript
// 当 T 是裸类型参数（naked type parameter）时触发分布式行为
type IsString<T> = T extends string ? true : false;

// 分布式展开：
// IsString<string | number>
// = (string extends string ? true : false) | (number extends string ? true : false)
// = true | false
// = boolean
```

**阻止分布式的技巧**：用元组包装类型参数

```typescript
type IsStringNoDistribute<T> = [T] extends [string] ? true : false;

// 不再分布式展开：
// IsStringNoDistribute<string | number>
// = [string | number] extends [string] ? true : false
// = false
```

### 逆变位置推断的原理

```typescript
// 函数的参数位置是逆变的（contravariant）
type Fn<T> = (arg: T) => void;

// 多个函数类型的联合 → 推断参数时取交集
type Union = Fn<{ a: string }> | Fn<{ b: number }>;
type Inferred = Union extends (arg: infer T) => void ? T : never;
// T = { a: string } & { b: number }
```

这背后的类型理论是：能被接受为 `Fn<A> | Fn<B>` 联合类型的实现函数，其参数必须同时满足 A 和 B 的约束，因此是交叉类型。

## 高频面试题解析

### Q1：如何判断两个类型是否相等？

**答案**：

```typescript
type IsEqual<A, B> =
  (<T>() => T extends A ? 1 : 2) extends
  (<T>() => T extends B ? 1 : 2)
    ? true
    : false;

type Test1 = IsEqual<string, string>;    // true
type Test2 = IsEqual<string, number>;    // false
type Test3 = IsEqual<any, string>;       // false（正确区分）
type Test4 = IsEqual<never, never>;      // true
```

这个实现利用了两个泛型函数类型的兼容性比较来精确判断类型相等，能正确处理 `any` 和 `never` 等边界情况。

### Q2：什么是分布式条件类型？使用时需要注意什么？

**答案**：
- 当条件类型的检查类型是**裸类型参数**（如 `T extends ...`）时，联合类型会自动分发
- 用 `[T]` 元组包装可以阻止分发
- 分布式是"功能"而非 bug，利用它可以遍历联合类型的每个成员

### Q3：如何实现 `DeepReadonly` 并处理数组和函数？

**答案**：

```typescript
type DeepReadonly<T> = T extends (...args: any[]) => any
  ? T  // 函数保持原样
  : T extends Array<infer U>
    ? ReadonlyArray<DeepReadonly<U>>  // 数组递归处理
    : T extends object
      ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
      : T;

type Test = DeepReadonly<{
  fn: () => void;
  items: { name: string }[];
  data: { nested: { value: number } };
}>;
// { readonly fn: () => void; readonly items: readonly { readonly name: string }[]; ... }
```
